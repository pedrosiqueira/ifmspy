Comunicação com o mundo exterior

Até agora, aprendemos algumas instruções em Python para realizar operações aritméticas. No entanto, nossos programas são como caixas fechadas - tudo o que acontece dentro deles permanece lá, sem interação com o mundo exterior. Para tornar nossos programas úteis, precisamos de uma maneira de fazê-los enviar e receber informações do mundo exterior. Isso pode incluir outros programas, arquivos que o programa lê ou escreve e, é claro, o usuário - a pessoa interagindo com o programa que desenvolvemos.

Portanto, um programa pode ser visto como uma caixa preta que recebe dados de entrada, os processa e produz uma saída. Para quem interage com o programa, não é necessário saber como ele realiza o processamento - basta fornecer os dados de entrada e esperar pela saída correta. Podemos comparar esse processo com a respiração: inspiramos os gases da atmosfera, nosso corpo os processa e exala os gases alterados.

Outra analogia comum é a culinária. Para fazer um bolo, seguimos uma receita (algoritmo) para combinar os ingredientes (entrada) e produzir um bolo (saída). Os ingredientes e o bolo final não fazem parte da receita em si - eles vêm de fora.

Saída de dados

O comando print é usado em Python para exibir informações no terminal. Dizemos "escrever no terminal" ou simplesmente escrever. Para usar o comando print, basta escrever print() e colocar o conteúdo que deseja exibir entre parênteses. Por exemplo:

print("Olá, mundo!")

Isso exibirá a mensagem "Olá, mundo!" no terminal.

Você também pode usar o comando print para exibir o valor de variáveis. Por exemplo:

x = 10
print(x)

Isso exibirá o valor da variável x, que é 10, no terminal.

Como uma expressão aritmética gera um resultado, você pode colocá-la dentro do print para exibir diretamente o resultado da expressão:

x = 10
print(x * 3 / 2 - x) # 5.0

Você pode exibir vários dados em um único comando print, separando-os por vírgulas. Neste caso, ele automaticamente insere um espaço entre os dados, exibindo tudo em uma única linha. Por exemplo:

x = 5
y = 10
print("A soma de", x, "e", y, "é", x + y)

Isso exibirá A soma de 5 e 10 é 15. Observe que o comando print inseriu espaços entre cada argumento para criar uma mensagem bem formatada.

O comando print é de uma categoria de comandos chamada de função. Pense em uma função como se fosse uma expressão aritmética, onde os operandos são os argumentos e os operadores são as vírgulas. Os argumentos de uma função sempre devem estar entre parênteses, precedidos pelo nome da função. E, semelhante a uma expressão aritmética, a maioria das funções também resulta em um valor, que pode ser atribuído a uma variável. Chamamos esse resultado de valor retornado ou simplesmente retorno. Entretanto, a função print é uma exceção, já que não resulta em um valor que possa ser atribuído a uma variável. Tomei a liberdade de adiantar um pouquinho o assunto de funções, mas, por hora, você não precisa entender a fundo esse conceito, o estudaremos mais para frente. Basta saber que certos comandos que usam parênteses são chamados de funções, que argumentos separados por vírgulas vão dentro dos parênteses das funções, e que as funções podem retornar valores.

A função print é uma ferramenta útil para exibir informações durante a execução de um programa e ajudar a entender como ele está funcionando.

Entrada de dados

O comando input é usado em Python para receber informações do teclado. Dizemos "ler do teclado" ou simplesmente ler. Para usar o comando input, basta escrever input() e atribuir esse comando a uma variável, que receberá o valor que o usuário digitar no teclado. Por exemplo:

x = input()

Este comando "congela" o programa, aguardando que o usuário digite algum valor e pressione a tecla Enter. Após isso, o que o usuário digitou será atribuído (armazenado) à variável x.

Para tornar a entrada de dados mais interativa, é possível solicitar ao usuário o dado esperado, inserindo uma mensagem entre os parênteses do input. Por exemplo:

nome = input("Qual é o seu nome? ")
print("Olá,", nome, "!")

Nesse caso, o programa solicitará ao usuário que informe seu nome, o qual será armazenado na variável nome. Em seguida, o programa exibirá uma saudação personalizada, usando o nome fornecido pelo usuário. Observe que inserimos um espaço no final da solicitação, para ela não ficar colada ao que o usuário digitar, apenas por questões de formatação.

O comando input também é uma função, mas diferentemente do print, só aceita um argumento, mas resulta em um valor, que é aquilo que o usuário digitou. Observe que o valor retornado pelo input é do tipo string, logo, os operadores aritméticos não funcionam como você talvez espere:

n1 = input('Digite um número: ')
n2 = input('Digite outro número: ')
soma = n1 + n2
print('A soma dos números digitados é', soma)

O operador de adição, quando operado em strings, realiza a operação de concatenação, que é "engatar" o final do primeiro operando no início do segundo operando.

Portanto, se você quiser que o Python interprete o retorno do input como um número inteiro, deve usar a função int para isso.

n1 = int(input('Digite um número: '))
n2 = int(input('Digite outro número: '))
soma = n1 + n2
print('A soma dos números digitados é', soma)

Da mesma forma, use a função float para interpretar uma string como um número real e a bool para interpretar como booleano:

numero1 = 3.4 # isso é um número!

numero2 = "2.5" # não se deixe enganar, isso não é um número!

numero3 = float(numero2) # agora sim, um número!

# claro, você pode usar float diretamente com o input
numero4 = float(input('Digite um número decimal: '))

acerto = numero1 + numero3 + numero4 # ótimo

erro = numero1 + numero2 # não vai dar bom...

Juntos, os comandos input e print podem ser usados para criar simples programas interativos que se comunicam com o usuário.

Mais um pouco sobre comentários

Computadores interpretam e executam códigos, mas eles não têm a capacidade de adivinhar tua intenção. Um simples erro no código, como não fechar um parêntese, e teu computador não saberá o que fazer.

Além disso, a programação muitas vezes é uma atividade colaborativa. Em uma empresa, o programa é desenvolvido em equipe. É essencial garantir que teu código seja correto tanto para a máquina quanto compreensível para outros programadores.

À arte de explicar o que teu código faz, damos o nome de documentação. Uma maneira comum de documentar sistemas é adicionar comentários diretamente no código-fonte…

Todas as linguagens de programação permitem que você insira comentários em teu código. Comentários são linhas que fazem parte do código, mas são ignoradas pelo computador. Eles servem exclusivamente para os programadores explicarem o propósito de determinadas partes do código.

Dica: tenha o hábito de adicionar comentários à medida que escreve o código.

Tipos de comentários

Há dois tipos de comentários: comentários simples e comentários em bloco.

Os comentários simples começam com uma cerquilha (#) e se estendem até o final da linha. Geralmente, são usados para pequenas anotações que cabem em uma única linha.

Os comentários em bloco começam com três aspas (simples ''' ou duplas """) e terminam com as mesmas três aspas (se você começou com as aspas simples, deve terminar com aspas simples). Eles são usados para escrever comentários mais extensos que ocupam várias linhas.

'''
Programa de exemplo de uso de comentários.
Autor: Pedro Henrique
Data: 08/08/2018
'''

# Solicita ao usuário que digite dois números
print("Digite dois números...")

# Lê os números que o usuário digita
a, b = map(int, input().split())

# Determina qual número é maior e qual é menor
maior = a if a > b else b
menor = b if a > b else a

# Escreve o maior e o menor número
print("Maior:", maior)
print("Menor:", menor)

Você pode não entender todos os comandos deste programa, mas ao ler os comentários, você pode ter uma ideia do que cada um deles está fazendo.

Principalmente em nossos primeiros programas, é comum incluir muitos comentários para ajudá-lo a se familiarizar com o Python.

Entretanto, tenha cuidado para não sobrecarregar teu código com comentários redundantes ou desnecessários, como o exemplo a seguir:

# Exibe "Gosto de programar"
print("Gosto de programar")

Dica: Comentários podem ser usados para testar teu código ou ignorar “temporariamente” um erro no teu código! Colocar cerquilha (#) no início de uma instrução fará com que o computador a ignore!

Quebrando linhas grandes

Ao contrário de outras linguagens que utilizam ponto e vírgula para marcar o final de uma instrução, em Python, geralmente, o final de uma instrução é determinado pelo final da linha. Isso significa que instruções extensas podem ocupar linhas longas. Para melhorar a legibilidade do código, é possível quebrar uma instrução extensa em várias linhas, inserindo uma barra invertida no ponto de quebra, indicando ao Python que a instrução continua na linha seguinte. Considere o seguinte código:

numero1 = 3.4
numero2 = 2.5
numero3 = 4.1
numero4 = 0.0

resultado = numero1 + numero2 + numero3 + numero4
print(resultado) # 10.0

O resultado é 10.0, como esperado. Entretanto, se você tentar quebrar a instrução da linha 6, o resultado não será exatamente o esperado:

resultado = numero1 + numero2
+ numero3 + numero4
print(resultado) # 5.9

Agora, resultado recebe apenas a soma de numero1 com numero2, pois numero3 e numero4 estão em outra linha, isto é, fazer parte de outra instrução. Para considerar essas duas linhas como uma mesma instrução, coloque uma barra invertida no final da primeira linha:

resultado = numero1 + numero2 \
          + numero3 + numero4
print(resultado) # 10.0

Agora sim, na segunda linha você pode até colocar mais espaços para alinhar o código e melhorar ainda mais a legibilidade.

Outra alternativa à barra invertida é envolver a instrução extensa entre parênteses, assim, o Python entenderá que a instrução só termina ao fechar os parênteses:

resultado = (numero1 + numero2
          +  numero3 + numero4)
print(resultado) # 10.0
