Tipos primitivos

Dados são valores brutos. Informação é o resultado da organização e interpretação desses dados de maneira significativa. Os computadores têm a capacidade de manipular e armazenar grandes volumes de dados com alta performance, permitindo que os seres humanos se dediquem a outras tarefas onde seu conhecimento é indispensável. Na programação, os dados são categorizados em diferentes tipos. Os tipos mais básicos de dados são chamados de tipos primitivos. A diferença entre tipos primitivos e outros tipos de dados, é que eles representam um único valor de dados. Por exemplo, o número 23 é primitivo, enquanto que uma data, como 23/07/2023, é composta por mais de um valor primitivo. Os tipos primitivos mais comuns são:

Inteiro: Tipo de dados que pertence ao conjunto dos números inteiros. Chamado de int, abreviação de inteiro. Exemplos:

  • Ele tem 3 irmãos.
  • A escada possui 8 degraus.
  • Está fazendo -3ºC hoje!

Real: Tipo de dados que pertence ao conjunto dos números reais. Chamado de float, abreviação de número de ponto flutuante, em alusão ao separador decimal que pode flutuar entre a direita e esquerda do número. Exemplos:

  • Ela tem 1,73 metro de altura.
  • Meu saldo bancário é de R$ 215,20.
  • No momento estou pesando 82,5 kg.

Lógico: Tipo de dados que pode assumir apenas duas situações, sim ou não, verdadeiro ou falso. Chamado de boolean, em homenagem a Boole, criador da álgebra booleana. Exemplos:

  • Ele foi aprovado.
  • A lâmpada está desligada.
  • A porta está trancada.

Caractere: Tipo de dados que representa uma unidade de informação que corresponde a um símbolo, como uma letra, um número ou um sinal de pontuação. Chamado de char, abreviação de character. Exemplo: a letra 'a', o dígito '3', ou o símbolo '#'. Observe, então, que 'oi' não é um caractere, mas uma combinação de dois caracteres, logo, não é um tipo primitivo. Chamamos de string a combinação de dois ou mais caracteres, que é um tipo de dados presente em todas as linguagens, inclusive Python, mas que não é tipo primitivo.

Variáveis

Uma variável é um espaço na memória do computador reservado para armazenar valores que são usados durante a execução de um programa. Como o nome indica, os valores armazenados em uma variável podem mudar ao longo do tempo. Por exemplo, você pode definir uma variável para armazenar a idade de uma pessoa e atualizá-la à medida que a pessoa envelhece.

Podemos imaginar a memória do computador como um armário cheio de gavetas de tamanho fixo. Cada gaveta pode armazenar um valor de um determinado tipo de dados. Se o valor for muito grande, pode ser necessário usar várias gavetas para armazená-lo. As gavetas representam as variáveis e o armário representa a memória do computador.

As variáveis são armazenadas na memória do computador enquanto o programa está em execução. Quando o programa termina, as variáveis deixam de existir e o espaço de memória que ocupavam fica disponível para ser usado por outros programas. Isso significa que um espaço de memória que foi usado por um programa para armazenar uma variável do tipo int pode ser usado posteriormente por outro programa para armazenar uma variável de um tipo diferente, como float.

Cada variável pode armazenar apenas um valor por vez. Por exemplo, você não pode armazenar dois números em uma única variável.

Você pode usar tantas variáveis quanto necessário em seu programa, desde que haja espaço suficiente na memória do computador para armazená-las. Como a memória do computador pode conter muitas variáveis, é importante diferenciá-las usando identificadores, que são nomes exclusivos atribuídos às variáveis.

Declarando variáveis

Declarar uma variável é solicitar ao computador um espaço na memória para armazenar um tipo de dados. Atribuir um valor à uma variável é armazenar um dado naquele espaço da memória reservado à sua variável. Em Python, a instrução (comando) para declarar uma variável é:

identificador = valor

Sendo identificador um nome exclusivo para sua variável, e valor o dado atribuído à variável. Exemplos:

idade = 30    # int
altura = 1.75 # float
destro = True # boolean
sexo = 'M'    # char

Observações:

  • O sinal de igual (=) é usado para atribuir um valor a uma variável.
  • O separador decimal em Python, bem como em praticamente todas as linguagens, é o ponto (.), e não a vírgula.
  • Variáveis do tipo boolean podem armazenar apenas dois tipos de valor, True, ou False.
  • O valor de variáveis do tipo char deve estar sempre entre aspas, para não serem confundidas com identificadores.
  • A cerquilha (#) é usada para indicar o início de um comentário no código. Comentários são textos que não são executados pelo computador e servem para fornecer explicações ou informações adicionais para quem está lendo o código.

Regras de identificação de variáveis:

  • Deve começar com uma letra.
  • Pode ter letras, números ou sublinhados depois do primeiro caractere.
  • Letras minúsculas são distinguidas de letras maiúsculas.
  • Não pode usar palavras reservadas da linguagem (como while, ou if).
  • Cada variável deve ter um identificador único.

Exemplos de identificadores válidos:

• meuDado • ano94 • nome_sobr • quant • Salario

Exemplos inválidos:

• 94ano • meu Dado • nome,sobr

Existem algumas convenções de declaração de identificadores. Dentre elas, as principais são: snake_case, que separa cada palavra por um sublinhado; camelCase que começa com a primeira palavra em minúsculo, e o início de cada próxima palavra em maiúsculo; e PascalCase, que deixa em maiúsculo apenas o início de cada palavra. No Python, a convenção adotada para variáveis é snake_case.

Dica: Escolha nomes significativos para suas variáveis. Quando os nomes das variáveis refletem claramente o que elas representam, é mais fácil entender o propósito e a lógica do código sem ter que ler comentários ou documentos adicionais. Isso também torna o código mais fácil de manter e modificar, por ser um código legível e compreensível.

Expressões aritméticas

Podemos usar operadores aritméticos para realizar operações com as nossas variáveis numéricas. Os operadores aritméticos em Python são:

Operador Descrição Exemplo
+ Adição x + y
- Subtração x - y
* Multiplicação x * y
/ Divisão x / y
// Divisão inteira x // y
% Módulo (resto da divisão) x % y
** Exponenciação x ** y

Você pode combinar tantos operadores, variáveis e valores quanto quiser, desde que seja uma expressão válida. Ao valores, também chamamos de literais ou constantes (o valor 10 sempre será 10, daí o nome constante ;p). Uma expressão aritmética sempre resulta em um valor. Você atribui o resultado de uma expressão aritmética a uma variável, caso contrário, ela acaba sendo uma instrução inócua, sem efeito prático:

p1 = 7.2
p2 = 8.3
t = 5
mp = (p1 + p2) / 2
mf = mp * 0.6 + t * 0.4

capital = 1400
taxa_de_juros = .07
tempo = 24
montante = capital * (1 + taxa_de_juros) ** tempo
juros = montante - capital

dividendo = 11
divisor = 4
quociente = dividendo // divisor
resto = dividendo % divisor

Em Python, não existe uma maneira nativa de definir uma constante, ou seja, um valor que não pode ser alterado durante a execução do programa. No entanto, é comum usar letras maiúsculas para nomear variáveis que devem ser tratadas como constantes e não devem ser alteradas. Por exemplo:

PI = 3.14

Neste exemplo, PI é uma variável que representa o valor de π. Embora tecnicamente seja possível alterar o valor de PI, a convenção de usar letras maiúsculas indica que essa variável deve ser tratada como uma constante e não deve ser alterada.

Em Python, você não pode utilizar diferentes tipos de dados em uma mesma expressão aritmética, com exceção de inteiros, reais, e booleanos. No caso, em expressões aritméticas, o valor booleano True é considerado como 1 e False como 0.

inteiro = 10
decimal = 5.4
booleano = True
caracter = 'b'

# você pode misturar esses tipos de dados apenas
válido = inteiro * decimal - booleano # 53.0

# você não pode misturar outros tipos de dados
inválido = caracter + inteiro # erro!

Prioridade e associatividade

A prioridade e a associatividade dos operadores determinam a ordem em que as operações são realizadas em uma expressão.

A prioridade dos operadores determina qual operador é executado primeiro se houver mais de um operador em uma expressão. Por exemplo, na expressão 2 + 3 * 4, a multiplicação tem prioridade mais alta do que a adição, então a multiplicação é realizada primeiro e o resultado é 14. Se quisermos que a adição seja realizada primeiro, podemos usar parênteses para alterar a ordem das operações: (2 + 3) * 4 resulta em 20.

A associatividade dos operadores determina a direção em que uma expressão é avaliada quando há vários operadores com a mesma prioridade. Por exemplo, na expressão 2 - 3 - 4, a subtração é associativa à esquerda, então a expressão é avaliada da esquerda para a direita e o resultado é -5. Se quisermos que a subtração seja realizada da direita para a esquerda, podemos usar parênteses para alterar a ordem das operações: 2 - (3 - 4) resulta em 3.

Se uma expressão contiver parênteses aninhados, como ((x / (y - w)) * z), a expressão no conjunto mais interno dentro dos parênteses (y - w, nesse caso) é avaliada primeiro.

Dica: Você pode usar parênteses redundantes para tornar a expressão mais clara. Por exemplo, na expressão 2 + (3 * 4), fica mais claro que a multiplicação deve ser realizada antes da adição.

Aqui está a tabela com os operadores vistos até o momento, ordenados pelo de maior prioridade para o de menor prioridade:

Operador Associatividade
( ) De dentro para fora
** Da direita para a esquerda
* / % // Da esquerda para a direita
+ - Da esquerda para a direita
= Da direita para a esquerda

Se não houver um operando à esquerda do + ou do -, eles são interpretados como operadores unários, isto é, operam em apenas um operando, diferentemente dos operadores aritméticos, que são binários. Neste caso, eles possuem maior prioridade que os operadores aritméticos. Exemplo:

x = 3
y = 4
result1 = x + -y  # -1
result2 = -x + y  # 1
result3 = -x * -y # 12

Já a associatividade dos operadores unários é da direita para a esquerda. Isto é, quando há vários operadores unários consecutivos, eles são aplicados da direita para a esquerda. Exemplo:

x = 3
result = - - - x # -3
# equivalente a:
# result = (- (- (- x)))

Passado, presente, futuro

Algoritmos seguem uma sequência de passos, um após o outro. Isso significa que, ao executar uma determinada instrução, todas as instruções anteriores já foram executadas, e a instrução atual não pode alterar as instruções anteriores. Além disso, as variáveis de tipos primitivos são independentes entre si. Isso quer dizer que, se você atribui o valor de uma variável x a outra variável y e, posteriormente, atribuir outro valor à variável y, a variável x permanecerá inalterada.

Portanto, para atualizar o valor de uma variável, você pode atribuí-la a si própria. Por exemplo:

x = 10
y = 20
z = x # neste momento, z vale 10
z = y # agora, z vale 20, mas x continua valendo 10
z = z * 2 # agora, z vale 40
