Operadores lógicos

Um bug muitas vezes entra por uma porta que você não sabia que deixou aberta.

Considere o seguinte trecho de código para verificar se uma idade está entre 18 e 64 anos:

if idade >= 18:
    if idade < 65:
        ... # aqui vai seu código

Embora funcione, instruções aninhadas não são muito legíveis. Utilizando operadores lógicos, podemos simplificar esse tipo de verificação.

Exemplos de operações lógicas:

  • SE estiver com sede E tiver água, ENTÃO beba!
  • SE tiver carro OU tiver carona, ENTÃO vá ao show!
  • SE NÃO tiver máscara, ENTÃO fique em casa!
  • SE tiver nota boa E NÃO faltou, ENTÃO aprovou!
  • SE NÃO tiver guarda-chuva E chover, ENTÃO espere a chuva passar!

Operador Conjuntivo

Para assegurar que duas condições são verdadeiras, utilize o operador and (E condicional):

if idade >= 21 and idade < 65:  ...

Apesar de opcional, parênteses redundantes podem tornar a expressão mais clara:

if (idade >= 21) and (idade < 65): ...

Operador Disjuntivo

Para assegurar que uma das duas ou ambas as condições são verdadeiras, utilize o operador or (OU condicional).

if horas > 40 or vendas > 25000.00: ...

Operador de negação

O operador unário not (NÃO lógico) “inverte” o valor de uma expressão lógica.

O operador de negação é colocado antes de uma condição.

if not masculino: ...

Diferentemente de outras linguagens de programação, no Python, o operador de negação tem menor prioridade que os operadores aritméticos. Portanto, as duas expressões seguintes são equivalentes:

if not (n % 2 == 0): ...
if not n % 2 == 0: ...

Na maioria dos casos, é possível evitar o operador de negação mudando o operador relacional e tornando a expresão mais legível:

if n % 2 != 0: ...

Qual seria a versão do seguinte trecho de código sem usar o operador not?

if not n <= 0: ...

Os operadores lógicos geram o valor True para verdadeiro e o valor False para falso.

if idade >= 21 and idade < 65:  ...
if horas > 40 or vendas > 25000.00: ...
if !masculino: ...

Uma condição que usa os operadores and ou or utiliza a avaliação em curto-circuito — a avaliação termina assim que encontra a resposta.

Considere a expressão

(i != 0) and (10 / i == 2)

A condição dependente (10 / i == 2) deve aparecer após o operador and para evitar a possibilidade de divisão por zero.

Suponha que um professor queira recompensar seus alunos que tenham bom desempenho e tenham perdido poucas aulas. Além disso, a recompensa exige que os alunos ingressem em três organizações escolares ou participem de duas atividades esportivas. A avaliação poderia ficar assim:

if nota > 9 and faltas <= 3 and (atividades >= 3 or esportes >=  2):
    ...

Para tornar o código mais legível, você pode envolver a condição do if entre parênteses e acrescentar quebras de linhas entre os operadores:

if (nota > 9 and faltas <= 3
    and (atividades >= 3 or esportes >= 2)):
    ...

    São bissextos todos os múltiplos de 4, exceto se for múltiplo de 100 mas não de 400, exemplo: 1996, 2000, 2004, 2008, 2012, 2016, 2020... Já 1900 não é bissexto, pois apesar de ser múltiplo de 4, não é múltiplo de 400 e é múltiplo de 100.

    Não são bissextos os demais anos.
