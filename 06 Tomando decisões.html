Estruturas de seleção

Em geral, as instruções em um programa são executadas sequencialmente, uma após a outra, seguindo a ordem em que são escritas. Esse fluxo é conhecido como estrutura sequencial.

No entanto, existem situações em que é necessário especificar que a próxima instrução a ser executada não seja necessariamente a próxima na sequência. Para lidar com essas situações, utilizamos a estrutura de seleção.

As instruções if e else tornam seus programas mais inteligentes, permitindo a eles tomar decisões baseadas em certas condições.

Você também aprenderá a usar os operadores relacionais, que servem para comparar valores, e são usados em conjunto com a instrução if.

As instruções if e else funcionam como na linguagem falada: se algo é verdadeiro, faça uma coisa; caso contrário, faça outra coisa.

  • SE eu ganhar na loteria, ENTÃO irei para a Itália; SENÃO, continuarei em 3L
  • SE chover, ENTÃO leve um guarda-chuva
  • SE tua nota for menor que 7, ENTÃO você ficou de recuperação; SENÃO, você foi aprovado

Operadores relacionais

Os operadores relacionais comparam dois valores. Você sempre coloca uma variável, literal ou expressão - ou uma combinação delas - em ambos os lados de um operador relacional.

Operador Descrição Exemplo
> Maior que numero > 10
< Menor que 10 * x < 8 – y
>= Maior ou igual a 9 >= 19.7
<= Menor ou igual a c <= 'e'
== Igual a i == j
!= Diferente de 'z' != 'g'

Um operador sempre resulta em um valor. Um operador relacional resulta no valor True quando o resultado da comparação é verdadeiro e resulta no valor False quando o resultado da comparação é falso.

ATENÇÃO:

  • Não confunda = com ==, lembre-se que um sinal de igual é atribuição e dois sinais de igual é comparação.
  • Somente valores equivalentes (mesmo tipo) devem ir em ambos os lados do operador relacional*. Ou seja, não compare um caractere com um número sem um typecast**!
  • Não inverta os caracteres dos operadores (=>, =<,=!, ..., é errado!).

*Com exceção, booleanos podem ser operados junto com números. Neste caso, o valor booleano True é equivalente ao número 1 e o valor booleano False é equivalente ao número 0.

**Typecast é um recurso da linguagem para converter um tipo de dado em outro.

A associatividade de operadores relacionais em Python é diferente de outras linguagens de programação. Por exemplo, x < y < z não significa nem (x < y) < z, nem x < (y < z), mas é equivalente a x < y and y < z, que usa o operador lógico and. Mais para frente estudaremos sobre operadores lógicos. Por hora, evite expressões com mais de um operador relacional.

Seleção simples

if condição:
    sequência de instruções
    que compõem o bloco do if

A condição é uma expressão cujo resultado deve ser booleano.

O dois-pontos é obrigatório.

Um bloco de código é uma sequência de instruções recuadas após o dois-pontos. Também chamamos o bloco de corpo.

Um recuo, ou indentação, se refere a uma certa quantidade de espaços ou tabulação que se coloca no início da instrução. Por convenção, no Python, se usa recuos de 4 espaços.

Caso a condição seja True, o corpo do if é executado. Caso contrário, o corpo do if não é executado, e o programa continua da próxima instrução depois do corpo do if.

Caso um bloco de código for composto por apenas uma linha, esta pode ficar logo após o dois-pontos.

Execute esse código e teste tanto com valores que atendam às condições do if quanto que não atendam. Procure entender o que está acontecendo no código:

salario = float(input("informe teu salario: "))
nota = float(input("informe tua nota: "))

Seleção composta

A instrução else é o irmão que complementa o if.

Todo else deve vir acompanhado de um if.

O corpo do if é executado caso a condição seja verdadeira. Caso a condição do if seja falsa, aí é o corpo do else que é executado.

Se existem vários níveis de recuo, cada nível deve ser recuado pela mesma quantidade adicional de espaço.

Execute esse código e teste com valores que explorem tanto o if quanto o else. Procure entender o que está acontecendo no código:

salario = float(input("informe teu salario: "))
nota = float(input("informe tua nota: "))
numero = int(input("informe teu numero da sorte: "))
idade = int(input("informe tua idade: "))

print("\nvoce ganha", salario)
if salario > 5000:
    print("vc ta bem ein!")
else:
    print("vc ainda chega la!")

print("\nvoce tirou nota", nota)
if nota < 7:
    print("vc esta de recuperacao!")
else: print("vc passou de ano!")

print("\nteu numero da sorte eh", numero)
if numero % 2 == 0: print("numero par")
else:
    print("numero impar")

if numero % 5 == 0: print("numero multiplo de 5.")
else: print("numero nao multiplo de 5")

print("\ntua idade eh", idade)
if idade >= 18:
    print("vc eh maior de idade,")
    print("ja eh responsavel legalmente.")
else:
    print("vc ainda eh menor de idade")

Apesar de eu ter feito variações de blocos na mesma linha e na linha de baixo, a convenção é sempre iniciar o bloco na linha de baixo, mesmo que seja composto por apenas uma linha.

Instruções aninhadas

Você pode colocar ifs e elses dentro de ifs e elses.

Quando colocamos instruções dentro de instruções, chamamos de instruções aninhadas.

Considere o seguinte programa: ler três números e escrever se o maior número é par ou ímpar.

Solução 1

print("digite 3 numeros...")
x = int(input())
y = int(input())
z = int(input())

if x > y:
    if x > z:
        print(x, "é o maior numero")
        if x % 2 != 0:
            print("e também é ímpar")
        else:
            print("e também é par")
    else:
        print(z, "é o maior numero")
        if z % 2 != 0:
            print("e também é ímpar")
        else:
            print("e também é par")
else:
    if y > z:
        print(y, "é o maior numero")
        if y % 2 != 0:
            print("e também é ímpar")
        else:
            print("e também é par")
    else:
        print(z, "é o maior numero")
        if z % 2 != 0:
            print("e também é ímpar")
        else:
            print("e também é par")

Observe que o if da linha 9 está dentro do if da linha 7 que por sua vez está dentro do if da linha 6. Isso quer dizer que para a linha 10 ser executada, os ifs das linhas 6, 7 e 9 devem ser verdadeiros.

Há várias formas de fazer um programa. Nesta, optei por usar ifs e elses. Daria para fazer apenas com ifs, porém ficaria mais complexo.

Solução 2

print("digite 3 numeros...")
x = int(input())
y = int(input())
z = int(input())

maior = 0  # Inicializando a variável 'maior'

if x > y:
    if x > z:
        maior = x
    else:
        maior = z
else:
    if y > z:
        maior = y
    else:
        maior = z

print(maior, "é o maior número")
if maior % 2 != 0:
    print("e também é ímpar")
else:
    print("e também é par")

Aqui está o mesmo programa feito de forma diferente. Para economizar linhas, armazenei o maior número numa variável e no final verifiquei se essa variável é par ou ímpar, ao invés de fazer a verificação várias vezes.

Solução 3

print("digite 3 numeros...")
x = int(input())
y = int(input())
z = int(input())

if x > y: maior = x
else: maior = y

if z > maior: maior = z

print(maior, "é o maior numero")
if maior % 2 != 0:
    print("e tambem é ímpar")
else:
    print("e tambem é par")

O mesmo exercício resolvido de mais outra forma. Dessa vez, usando ainda menos if-else’s.

Atenção para a indentação!

Neste exemplo, o else corresponde ao if logo antes dele:

if x > 5:
    if y > 5:
        print("x e y são maiores que 5")
    else:
        print("x é maior que 5")

Já neste exemplo, o else corresponde ao if mais distante:

if x > 5:
    if y > 5:
        print("x e y são maiores que 5")
else:
    print("x NÃO é maior que 5")

Seleção encadeada

Considere o seguinte exercício:

A nota final de um estudante é calculada a partir de três notas atribuídas, respectivamente, a um trabalho, a um seminário e a uma prova. A média das três notas mencionadas obedece aos pesos da tabela A. Dadas as três notas de um estudante, qual sua média ponderada e seu conceito conforme a tabela B?

Tabela A
Nota Peso
Trabalho 2
Seminário 3
Prova 5
Tabela B
MÉDIA PONDERADA CONCEITO
Maior ou igual a 8,5 A, aprovado :D
Maior ou igual a 7 e menor que 8,5 B, aprovado :)
Maior ou igual a 5 e menor que 7 C, recuperação :|
Maior ou igual a 2,5 e menor que 5 D, recuperação :(
Menor que 2,5 E, reprovado :O

Solução 1:

print("Boletim Digital")
print("###############\n")

t = float(input("Trabalho: "))
s = float(input("Seminário: "))
p = float(input("Prova: "))
m = (t * 2 + s * 3 + p * 5) / 10

print("\nMédia:", m)

if m >= 8.5:
    print("Conceito A, aprovado :D")
else:
    if m >= 7:
        print("Conceito B, aprovado :)")
    else:

            print("Conceito C, recuperação :|")
        else:
            if m >= 2.5:
                print("Conceito D, recuperação :(")
            else:
                print("Conceito E, reprovado :O")

Embora esse código funcione, ele cria um aninhamento desnecessário de instruções if e else, o que torna o código menos legível e mais difícil de manter. A utilização do aninhamento excessivo pode ser evitada usando a estrutura elif.

As instruções combinadas if, elif e else são mutuamente exclusivas. Isto é, apenas o corpo da primeira instrução verdadeira é executado. Se nenhuma for verdadeira, e houver um else, apenas o corpo do else é executado.

Operador ternário

Em Python, assim como em algumas outras linguagens de programação, você pode usar o operador ternário para expressar uma operação condicional de forma mais concisa. O operador ternário é uma forma compacta de escrever um if-else de uma linha.

A sintaxe básica do operador ternário em Python é a seguinte:

valor_se_verdadeiro if condicao else valor_se_falso

A condicao é a expressão que será avaliada. Se for verdadeira, o resultado da expressão é valor_se_verdadeiro; caso contrário, o resultado da expressão é valor_se_falso.

Aqui está um exemplo de código que utiliza um if-else para determinar o maior entre dois números:

a = 10
b = 5

maior = a if a > b else b
print(maior)

Neste caso, o operador ternário a if a > b else b verifica se a é maior que b. Se for verdadeiro, resulta no valor de a; caso contrário, resulta no valor de b. Isto é equivalente a escrever:

a = 10
b = 5

if a > b:
    maior = a
else:
    maior = b

print(maior)

Considere também este exemplo:

nota = float(input("Qual é a tua nota? "))
print("Aprovado!" if nota >= 6 else "Reprovado!")

O operador ternário é útil para expressões condicionais simples, mas deve ser usado com moderação para não comprometer a legibilidade do código.

Valores Verdadeiros e Falsos em Python

Enquanto os valores booleanos True e False correspondem diretamente aos números 1 e 0, respectivamente, o Python oferece uma abordagem flexível para considerar outros valores como verdadeiros ou falsos em contextos condicionais. Isso acontece quando esses valores são avaliados de forma isolada dentro de estruturas como if ou while, sem o uso de operadores relacionais, ou quando são passados como argumentos para a função bool().

Considere o exemplo abaixo, em que a variável var é avaliada isoladamente no contexto do if:

var = -1
if var:
    print(var, "é verdadeiro")
else:
    print(var, "é falso")

var = 0
if var:
    print(var, "é verdadeiro")
else:
    print(var, "é falso")

var = 3
if var:
    print(var, "é verdadeiro")
else:
    print(var, "é falso")

var = "0"
if var:
    print(var, "é verdadeiro")
else:
    print(var, "é verdadeiro")

Veja o código equivalente usando operador ternário:

var = -1
print(var, "é", "verdadeiro" if var else "falso")

var = 0
print(var, "é", "verdadeiro" if var else "falso")

var = 3
print(var, "é", "verdadeiro" if var else "falso")

var = "0"
print(var, "é", "verdadeiro" if var else "falso")

Quando avaliados isoladamente em um contexto condicional, os seguintes valores são considerados falsos:

  • As constantes None e False,
  • O número zero,
  • Objetos cujo comprimento é zero, como uma string vazia "",
  • Objetos cujo método __bool__ retorna False.

Para todos os outros valores, a avaliação resulta em True, inclusive para a string "False". Apesar de seu conteúdo ser "False", ela não está vazia, portanto, é considerada True.

Erros

Um erro de sintaxe ocorre quando o código que você escreve não segue a gramática definida pela linguagem de programação. Esses erros são detectados durante a análise do código, antes de ser executado. Eles geralmente resultam em mensagens de erro que indicam a linha e o local onde a violação ocorreu. Erros de sintaxe incluem coisas como escrever palavras-chave erradas, omitir parênteses de fechamento, usar operadores de forma incorreta e outros problemas que violam as regras da linguagem.

Quantos erros de sintaxe você consegue encontrar a seguir?

nota1 = float(input("qual tua nota? ))
if nota2 >= 6
print("aprovado"
else:
    print(reprovado)

Erros de sintaxe também podem ser chamados de erros em tempo de compilação ou simplesmente erros de compilação, porque eles são detectados pelo compilador antes da execução do programa, impedindo o programa de ser executado, mesmo que apenas algumas instruções estejam erradas.

Um erro em tempo de execução ou simplesmente erro de execução ocorre quando o programa está em execução e encontra uma situação imprevista que impede seu funcionamento normal. Isso pode ser causado por vários motivos, como entradas inválidas, tentativas de acesso a variáveis inexistentes, divisões por zero, entre outros. Esses erros são frequentemente chamados de exceções e interrompem a execução do programa a partir de onde ocorreram, geralmente exibindo uma mensagem de erro para o usuário.

Tente digitar uma palavra em vez de um número para ver um exemplo de erro de execução:

print("Programa para calcular o quadrado de um número...")
numero = int(input("Digite um número: "))
quadrado = numero * numero
print("O quadrado do número é", quadrado)

Um erro de lógica ocorre quando um programa produz resultados indesejados devido a problemas em sua lógica interna ou na implementação do algoritmo. Esse tipo de erro não é detectado pelo interpretador ou compilador como um erro, porque o código está sintaticamente correto e não resulta em falhas de compilação ou execução. No entanto, o programa pode não realizar as operações desejadas ou pode produzir resultados incorretos.

Considere o seguinte programa que não calcula corretamente o quadrado de um número, resultando em um erro de lógica:

print("Programa para calcular o quadrado de um número...")
numero = int(input("Digite um número: "))
quadrado = numero * 2
print("O quadrado do número é", quadrado)
