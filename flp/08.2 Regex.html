8.2 Expresses regulares

8.2.1 Introduo

Uma expresso regular (regex) uma sequncia de caracteres que especifica um padro de busca. regex uma linguagem formal implementada em vrias linguagens. Apesar de regex no ser orientado a objetos, o Python possui vrios objetos para trabalhar com regex. O mdulo re possui vrias classes e funes para trabalhar com regex. vamos comear com a funo findall:

>>> import re
>>> len(re.findall("e", "Python 3.10 was released on October 04, 2021"))
4
>>> re.findall("\\d", "Python 3.10 was released on October 04, 2021")
['3', '1', '0', '0', '4', '2', '0', '2', '1']
>>> re.findall(r"\d", "Python 3.10 was released on October 04, 2021")
['3', '1', '0', '0', '4', '2', '0', '2', '1']
>>> re.findall("\d", "Python 3.10 was released on October 04, 2021")  
<stdin>:1: SyntaxWarning: invalid escape sequence '\d'
['3', '1', '0', '0', '4', '2', '0', '2', '1']
>>> re.findall(r"z", "Python 3.10 was released on October 04, 2021")  
[]

O primeiro argumento da funo findall o padro de correspondncia e o segundo argumento a string de busca. O retorno uma lista com as correspondncias encontradas. Na primeira invocao, buscada pela letra "e". Na segunda invocao buscado por um dgito, que no regex, representado pelo caractere especial \d. Como numa string a barra invertida usada como sequncia de escape, precisamos escapar a barra invertida, colocando outra barra invertida, para no interpretar \d como sequncia de escape. Regex possui vrios caracteres especiais para formar padres de texto. Para no ter que ficar escapando cada barra invertida, o Python possui a string bruta, ou r-string, que automaticamente escapa qualquer barra invertida, sem a necessidade de pr a segunda barra invertida, como na terceira invocao.

Para ilustrar a diferena de string e r-string, veja como uma sequncia de escape no interpretada numa r-string: 
>>> print("Hello\nWorld\t!")  # Interpretando sequncia de escape
Hello
World   !
>>> print("Hello\\nWorld\\t!")  # Ignorando manualmente
Hello\nWorld\t!
>>> print(r"Hello\nWorld\t!")  # Ignorando automaticamente
Hello\nWorld\t!

Apesar da quarta invocao funcionar, exibida uma mensagem de aviso, pois no ignoramos a barra invertida. Isso funciona pois at a verso 3.12, que estou usando, sequncias de escape invlidas so automaticamente ignoradas e a barra invertida tratada como literal. Mas est previsto para alguma prxima verso o aviso se tornar em exceo SyntaxError e esses cdigos legados pararem de funcionar. Portanto, sempre que usar um caractere regex especial, que contenha uma barra invertida, no se esquea de adicionar uma segunda barra invertida ou usar r-string.

As principais funes do mdulo re so:

Funo Descrio
match(pattern, string, flags=0) Busca o padro no incio da string, retornando um objeto Match.
search(pattern, string, flags=0) Busca a primeira ocorrncia do padro na string, retornando um objeto Match.
findall(pattern, string, flags=0) Busca por todas as ocorrncias do padro na string, retornando uma lista de strings ou de tuplas, dependendo do caso.
finditer(pattern, string, flags=0) Busca por todas as ocorrncias do padro na string, mas retorna um iterador de Matchs.
sub(pattern, repl, string, count=0, flags=0) Substitui todas as ocorrncias do padro na string por outra substring.
split(pattern, string, maxsplit=0, flags=0) Divide a string em uma lista de strings que correspondam ao padro.

Os principais mtodos de Match so group, que retorna a string encontrada, start e end, que retornam, respectivamente, o incio e o fim-1 onde encontrou a string.

Um objeto iterador, como veremos no prximo captulo, considerado "lazy" porque os elementos so iterados sob demanda, ou seja, apenas quando solicitados. Isso significa que o iterador no gera todos os elementos imediatamente, o que pode economizar memria e processamento.

Por outro lado, uma lista considerada "eager" porque todos os seus elementos so gerados de uma vez, ocupando memria e processamento desde o incio. Por isso que iteradores no possuem ndices nem possvel invocar a funo len, pois s sabemos a quantidade de elementos depois de ter percorrido toda a sequncia.

Toda vez que voc invoca essas funes, o Python "compila" o padro de busca em uma estrutura interna para realizar a busca, o que leva tempo. Para maior eficincia, se um padro for reutilizado diversas vezes, melhor realizar essa compilao apenas uma vez com a funo compile:

def search(pattern_string, string_list):
    pattern = re.compile(pattern_string)  # Compila o padro de expresso regular uma vez fora do loop

    results = []
    for string in string_list:
        results.append(pattern.findall(string))  # aqui o padro j est compilado
    return results

A funo compile retorna um objeto Pattern, que possui o padro regex compilado e mtodos com os mesmos nomes das funes anteriores para realizar buscas utilizando o padro compilado. Embora seja importante cuidar da eficincia, o compilador Python j oferece ferramentas para aprimorar cdigos passveis de melhoria. Ele identifica partes que poderiam ser reescritas de maneira mais eficiente, como no exemplo anterior, se no tivssemos utilizado a funo compile. O Python automaticamente compila um padro e o armazena em cache para uso posterior, otimizando assim o desempenho.

8.2.2 Conjuntos de caracteres

Regex possui padres para representar conjuntos de caracteres, tambm chamados de classes de caracteres. So eles:

[aeiou]  [a-z]

Corresponde a qualquer um dos caracteres dentro dos colchetes.

>>> re.findall("[aeiou]", "quais as vogais dessa frase?")
['u', 'a', 'i', 'a', 'o', 'a', 'i', 'e', 'a', 'a', 'e']

Se contiver um hfen entre dois caracteres, corresponde ao intervalo fechado de caracteres entre eles.

>>> re.findall("[j-o]", "letras entre 'j' e 'o' so essas...")
['l', 'n', 'j', 'o']

Voc pode colocar vrios conjuntos dentro dos colchetes.

>>> re.findall("[A-Z0-9]", "S qUero as Maisculas e os nm3r05.")
['S', 'U', 'M', '3', '0', '5']

Para corresponder ao hfen tambm, coloque-o no incio ou final dos colchetes.

>>> re.findall("[aeiou0-9-]", "Hfen (-) e dgitos 0 e 8.")
['e', '-', 'e', 'i', 'o', '0', 'e', '8']

Na tabela Unicode, letras acentuadas tm cdigos diferentes.

>>> re.findall("[---]", "quele PSSEGO piau pde Mller")
['', '', '', '', '']

[^aeiou]  [^a-z]

Corresponde a qualquer caractere que no seja os que esto dentro dos colchetes.

>>> re.findall("[^aeiou ]", "Vogais aqui no")
['V', 'g', 's', 'q', 'n', '']
>>> re.findall("[^0-9]", "35T3 P3QU3N0 T3XT0")
['T', ' ', 'P', 'Q', 'U', 'N', ' ', 'T', 'X', 'T']

Existem vrios caracteres especiais em regex. Mas dentro dos colchetes, os nicos especiais so -, ^ e \. Para escap-los, voc tambm pode preced-los com uma \.

.

Corresponde a qualquer caractere exceto quebra de linha.

>>> re.findall(".e", "Voc me ligou naquela tarde vazia\ne me valeu o dia")
['me', 'ue', 'de', 'me', 'le']

O . dentro dos colchetes funciona como ponto literal. Exemplo que captura cada caractere seguido por uma pontuao:

>>> re.findall(r".[.,?!]", "Pare! Repita? Oras, bolas.")
['e!', 'a?', 's,', 's.']

Exemplo que captura vogais distantes por dois caracteres:

>>> re.findall(r"[aeiou]..[aeiou]", "Processando erros fatais.")
['essa', 'erro', 'atai']

\d

Corresponde a qualquer dgito. Abreviao de [0-9].

>>> re.findall(r"\d\d", "Dia de So Valentim: 14/2/270.")
['14', '27']

\w

Corresponde a qualquer caractere alfanumrico (de qualquer idioma) ou sublinhado.

>>> "".join(re.findall(r"\w", "1 ao! 2_? 3-."))
'1ao2_3'

\s

Corresponde a qualquer caractere de espao em branco (whitespace).

>>> len(re.findall(r"\s","Quantos\nwhitespaces essa\tfrase\rpossui?"))
4

\D  \W  \S

Se voc colocar em maisculos, a negao do significado em minsculo. Isto , \D, \W e \S correspondem, respectivamente, a qualquer caractere no-dgito, no alfanumrico nem sublinhado, no whitespace.

>>> re.sub(r"\D", "", "S quero os nmeros deste telefone: +55(67)3012-4567")
'556730124567'
>>> re.sub(r"\W", "", "Apenas L3TR45! Juntas, misturadas.")
'ApenasL3TR45Juntasmisturadas'
>>> len(re.findall(r"\S","Quantidade de caracteres,\ndesconsiderando whitespaces\n"))
49

Sequncias de escape

Se voc quiser escapar um caractere especial, para trat-lo literalmente, basta preced-lo por uma barra invertida:

>>> re.findall("\d\.\d", "1.2, 3.4, 5.6 e, por fim, 7.8")
['1.2', '3.4', '5.6', '7.8']

Regex tambm interpreta sequncias de escape. Portanto, uma string usada como padro de busca compilada duas vezes, uma pelo prprio Python e outra pela engine de regex do Python. Isso quer dizer que se voc colocar a string \\n como padro de busca, primeiramente o Python entender que para escapar a barra invertida, tornando a string na barra invertida literal seguida pela letra n, que o regex ainda entender como uma quebra de linha.

Considere cada findall seguinte. O primeiro argumento um padro regex, que interpretado tanto pelo Python quanto pela engine regex, j o segundo argumento interpretado apenas pelo Python.

>>> print("a\nb")
a
b
>>> print("a\\nb")
a\nb
>>> re.findall(r"a\nb", "a\nb")
['a\nb']
>>> re.findall("a\\nb", "a\nb")
['a\nb']
>>> re.findall("a\\nb", "a\\nb")
[]

Na linha 6, o Python interpreta o padro \n como quebra de linha, por isso correspondeu string de busca. Na linha 8, o Python interpreta o padro \\n como \ e n, que a engine tambm interpreta como quebra de linha, por isso da correspondncia. Na linha 10, o padro o mesmo da linha 8, mas como a string de busca interpretada apenas pelo Python, o \\n se torna \ e n, que no corresponde quebra de linha do padro.

Se por um lado o Python ainda permite sequncias de escape invlidas, tratando a barra invertida literamente, por outro lado, a engine regex tambm permite caracteres especiais invlidos, mas descarta a barra invertida. Por exemplo, \_ no nem uma sequncia de escape, nem um caractere especial regex. E tanto \_ quanto \\_ so interpretados pelo Python como \_, que por outro lado, interpretado pela engine como _ apenas. J pelo Python, \\\_ vira \\_, que, pela engine, vira \_.

import re
print("a\_b")                         # a\_b
print("a\\_b")                        # a\_b
print("a\\\_b")                       # a\\_b
print(re.findall("a\_b", "a_b"))      # ['a_b']
print(re.findall("a\\_b", "a_b"))     # ['a_b']
print(re.findall("a\_b", "a\_b"))     # []
print(re.findall("a\\_b", "a\\_b"))   # []
print(re.findall("a\\\_b", "a_b"))    # []
print(re.findall("a\\\_b", "a\_b"))   # ['a\\_b']
print(re.findall("a\\\_b", "a\\_b"))  # ['a\\_b']
print(re.findall("a\\\_b", "a\\\_b")) # []

Observao: Voc pode estar se perguntando por que as linhas 10 e 11 imprimiram as strings como a\_b em vez de a\_b, conforme as linhas 2 e 3. Isso ocorre porque ao imprimir listas, as sequncias de escape no so interpretadas.

>>> l = ["string normal", "sequncias\nde\tescape"]
>>> print(l)
['string normal', 'sequncias\nde\tescape']
>>> print(f"['{l[0]}', '{l[1]}']")
['string normal', 'sequncias
de      escape']

8.2.3 Quantificadores

Quantificadores indicam a quantidade de caracteres ou expresses a corresponder.

*

Corresponde ao seu predecessor zero ou mais vezes.

>>> re.findall(r"fo*", "foo bar. f b. fooooo barrrrr")
['foo', 'f', 'fooooo']
>>> re.findall(r"\d*\.\d*", "num1 = 123\nnum2 = 34.56\nnum3 = .7\nnum4 = 8.901")
['34.56', '.7', '8.901']

*

Corresponde ao seu predecessor uma ou mais vezes.

>>> re.findall(r"fo+", "foo bar. f b. fooooo barrrrr")
['foo', 'fooooo']
>>> re.findall(r"\d+\.\d+", "num1 = 123\nnum2 = 34.56\nnum3 = .7\nnum4 = 8.901")
['34.56', '8.901']

{n}

Corresponde ao seu predecessor exatamente n vezes.

>>> list(re.finditer(r"o{3}", "foobar, foooobar, foooooobar"))
[<re.Match object; span=(9, 12), match='ooo'>, <re.Match object; span=(19, 22), match='ooo'>, <re.Match object; span=(22, 25), match='ooo'>]

{n,}

Corresponde ao seu predecessor pelo menos n vezes.

>>> list(re.finditer(r"o{3,}", "foobar, foooobar, foooooobar"))
[<re.Match object; span=(9, 13), match='oooo'>, <re.Match object; span=(19, 25), match='oooooo'>]

x{m,n}

Corresponde ao seu predecessor pelo menos m e no mximo n vezes, sendo 0 ≤ m < n.

>>> re.findall(r"\d{3,5}", "Capturando entre 3 e 5 dgitos: 1 23 456 7890 12345 678901")
['456', '7890', '12345', '67890']

Observe que no exemplo anterior, apesar de 678901 ter 6 dgitos, seus 5 primeiros dgitos ainda foram capturados.

?

Corresponde ao seu predecessor zero ou uma vez.

>>> re.findall(r"vultu?osa", "Possui vultuosa quantidade de dinheiro. Precisa de uma mala vultosa para guard-lo.")
['vultuosa', 'vultosa']

*?  +?  ??  {n}?  {n,}?  {n,m}?

Por padro, os quantificadores so greedy (gulosos), isto , eles tentar corresponder o mximo possvel de seu predecessor. Ao colocar um ? aps o quantificador, ele se torna non-greedy, isto , ele tenta corresponder o mnimo possvel de seu predecessor.

Suponha que queiramos extrair os ids de todos os elementos html de uma pgina.

html_content = """...
<h1 id="ttulo">Ttulo</h1>
...
<a id="voltar" href="../">Voltar</a>
..."""

print(re.findall(r"id=\".*\"", html_content))
# Sada: ['id="ttulo"', 'id="voltar" href="../"']

O primeiro id foi extrado conforme o esperado, mas o segundo id, no.

Funcionamento de quantificadores greedy

O regex comea avanando a string de busca at se deparar com o i.

Ao encontrar o i, ele continua avanando medida que demais caracteres do padro correspondem string.

Os prximos caracteres do padro so .*, que significa "tudo at o final da linha", logo, o regex avana at o final da linha, capturando tudo.

O ltimo caractere do padro \", assim o regex sabe que a correspondncia ter que terminar com aspas. Portanto, o regex precisa voltar pela string at encontrar as aspas. Esse passo conhecido como backtracking (retrocesso).

Para alterar esse comportamento, especificamos o quantificador como non-greedy, colocando um ? aps ele:

print(re.findall(r"id=\".*?\"", html_content))
# Sada: ['id="ttulo"', 'id="voltar"']

8.2.2 Obtendo informaes de expresses regulares

Instead of returning a list of match objects, as you would expect, it returns a list of matching strings, or tuples. Sometimes it's strings, sometimes it's tuples. It's not a very good API at all! As with all bad APIs, you'll have to memorize the differences and not rely on intuition. The type of the return value depends on the number of bracketed groups inside the regular expression: If there are no groups in the pattern, re.findall will return a list of strings, where each value is a complete substring from the source string that matches the pattern If there is exactly one group in the pattern, re.findall will return a list of strings where each value is the contents of that group If there are multiple groups in the pattern, re.findall will return a list of tuples where each tuple contains a value from a matching group, in order When you are designing function calls in your own Python libraries, try to make the function always return a consistent data structure. It is often good to design functions that can take arbitrary inputs and process them, but the return value should not switch from a single value to a list, or a list of values to a list of tuples depending on the input. Let re.findall be a lesson! The examples in the following interactive session will hopefully clarify the differences: >>> import re >>> re.findall('a.', 'abacadefagah') ['ab', 'ac', 'ad', 'ag', 'ah'] >>> re.findall('a(.)', 'abacadefagah') ['b', 'c', 'd', 'g', 'h'] >>> re.findall('(a)(.)', 'abacadefagah') [('a', 'b'), ('a', 'c'), ('a', 'd'), ('a', 'g'), ('a', 'h')] >>> re.findall('((a)(.))', 'abacadefagah') [('ab', 'a', 'b'), ('ac', 'a', 'c'), ('ad', 'a', 'd'), ('ag', 'a', g'), ('ah', 'a','h')]
