Quando objetos so semelhantes

Boas prticas de programao incluem no duplicar cdigo. Uma das formas de reaproveitar cdigo atravs de herana.

Herana bsica

Todas as classes em Python herdam implicitamente da classe especial object, que possui alguns mtodos incorporados.

A classe herdada chamada de superclasse ou pai. A classe que herda chamada de subclasse ou filha. Herdar uma classe tambm chamado de estender uma classe.

Basicamente, herana usada para adicionar ou alterar funcionalidades em uma classe.

Vamos aplicar o conceito em um aplicativo gerenciador de contatos.

class Contact:
    all_contacts = []

    def __init__(self, name, email):
        self.name = name
        self.email = email
        Contact.all_contacts.append(self)

A varivel all_contacts uma varivel de classe. Semelhante a uma varivel global, h apenas UMA varivel de classe Contact.all_contacts compartilhada por todo o sistema. Para acessar uma varivel de classe, informe o nome da classe antes do nome da varivel.

Se teu sistema possuir contatos "fornecedores", os quais podero receber pedidos, no seria interessante implementar um mtodo de pedidos diretamente na classe Contact, pois isso permitiria a contatos que no so fornecedores tambm receberem pedidos:

class Supplier(Contact):
        def order(self, order):
            print(f"If this were a real system we would send '{order}' order to '{self.name}'")

Tente instanciar objetos Contact e Supplier e chamar o mtodo order para ver o que acontece.

Extendendo tipos incorporados

Podemos estender a funcionalidade de listas em Python para buscar por um item especfico:

class ContactList(list):
    def search(self, name):
        """Return all contacts that contain the search value in their name."""
        matching_contacts = []
        for contact in self:
            if name in contact.name:
                matching_contacts.append(contact)
        return matching_contacts

Faa a seguinte alterao na classe Contact:

class Contact:
    all_contacts = ContactList() # altere aqui

    def __init__(self, name, email):
        self.name = name
        self.email = email
        Contact.all_contacts.append(self)

Lembrando, em Python, podemos inicializar listas com [], que um atalho para list().

Outros tipos incorporados do Python so object, list, set, dict, file, int e float, que podem ser herdados para adicionar funcionalidades.

Sobrescrita e super

Alm de adicionar funcionalidades, herana tambm pode alterar funcionalidades. Vamos sobrescrever o construtor de Contact:

class Friend(Contact):
        def __init__(self, name, email, phone):
            super().__init__(name, email) # chama o mtodo __init__ pai
            self.phone = phone

A funo super usada dentro de uma classe para se referir classe pai e no pode ser usada fora do escopo de uma classe. Ela acessa os atributos e mtodos declarados na classe pai, til quando sobrescrevemos um mtodo na classe filha.

Herana mltipla

Herana mltipla envolve uma subclasse herdando de mltiplas superclasses, obtendo acesso s funcionalidades de cada uma. Embora o conceito parea simples, muitas vezes menos prtico do que o esperado, levando muitos programadores experientes a desaconselhar o seu uso.

O mixin a forma mais simples e til de herana mltipla. Ele uma superclasse destinada a ser herdada por outras classes para adicionar funcionalidade extra. Por exemplo, ao adicionar a capacidade de enviar emails classe Contact, podemos criar um mixin que lida com o envio de emails:

class MailSender:
    def send_mail(self, message):
        print("Sending mail to " + self.email)
        # Add e-mail logic here

Embora o mixin por si s no tenha uma funo especial (nem pode ser uma classe independente funcional), ele permite a criao de uma nova classe que combina caractersticas de Contact e MailSender, usando herana mltipla:

class EmailableContact(Contact, MailSender):
    pass

Vamos considerar alternativas ao mixin:

  • Herana simples: Poderamos adicionar a funo send_mail subclasse. No entanto, isso resultaria em duplicao da funcionalidade de email para outras classes que tambm precisassem disso.
  • Funo independente: Poderamos criar uma funo separada para enviar emails e cham-la com o endereo de email correto quando necessrio, evitando duplicao de cdigo.
  • Composio: Em vez de herana, poderamos usar composio, onde a classe EmailableContact teria um objeto MailSender como atributo.
  • Monkey patching: Poderamos adicionar um mtodo send_mail classe Contact aps sua criao, recurso esse que veremos mais tarde.

Herana mltipla eficaz para combinar mtodos de diferentes classes, mas se complica ao lidar com chamadas de mtodos de mesmo nome em vrias superclasses.

Para explorar essas questes, consideremos a adio de um endereo residencial classe Friend. Existem vrias abordagens possveis: passar os elementos do endereo como parmetros para o construtor da classe Friend, armazen-los em uma estrutura (como uma tupla ou dicionrio) e pass-los como um nico argumento ou at mesmo criar uma classe separada chamada Address para encapsular esses elementos e passar uma instncia dela ao construtor da classe Friend.

Embora a composio seja uma soluo eficaz para esse cenrio, a herana tambm possvel. Neste contexto, criamos uma nova classe chamada AddressHolder:

class AddressHolder:
    def __init__(self, street, city, state, code):
        self.street = street
        self.city = city
        self.state = state
        self.code = code

Esta classe simplesmente armazena os elementos do endereo como variveis de instncia durante a inicializao.

O problema do diamante

Alterar Friend para herdar de AddressHolder gera ambiguidade ao chamar o mtodo super().__init__. De qual superclasse estaramos nos referindo? Poderamos resolver da seguinte forma:

class Friend(Contact, AddressHolder):
    def __init__(
        self, name, email, phone, street, city, state, code):
        Contact.__init__(self, name, email) # chame o construtor do pai Contact
        AddressHolder.__init__(self, street, city, state, code) # chame o construtor do pai AddressHolder
        self.phone = phone

O problema acontece quando ambas as superclasses tambm herdam de uma classe em comum, formando um diamante (ou losango) no diagrama de classes:

Nesse exemplo, como todas as classes herdam de object, o mtodo __init__ de object seria chamado duas vezes. Como este no faz nada, aqui, no teria problema. Mas imagine tentar se conectar ao banco de dados duas vezes para cada requisio! Portanto, a classe base deveria ser chamada apenas uma vez. Mas por quem? Em qual ordem? Friend Contact Object?

Considere mais um exemplo do problema do diamante:

Vamos codigar esse diagrama:

class BaseClass:
    num_base_calls = 0

    def call_me(self):
        print("Calling method on Base Class")
        self.num_base_calls += 1


class LeftSubclass(BaseClass):
    num_left_calls = 0

    def call_me(self):
        BaseClass.call_me(self)
        print("Calling method on Left Subclass")
        self.num_left_calls += 1


class RightSubclass(BaseClass):
    num_right_calls = 0

    def call_me(self):
        BaseClass.call_me(self)
        print("Calling method on Right Subclass")
        self.num_right_calls += 1


class Subclass(LeftSubclass, RightSubclass):
    num_sub_calls = 0

    def call_me(self):
        LeftSubclass.call_me(self)
        RightSubclass.call_me(self)
        print("Calling method on Subclass")
        self.num_sub_calls += 1

Instancie um objeto de Subclass e chame seu mtodo call_me. Observe que call_me da classe base chamado duas vezes. Isso pode ser um problema. Imagine um aplicativo de banco realizando o depsito duas vezes!

Substitua as chamadas explcitas aos mtodos call_me das superclasses por apenas uma chamada super().call_me(). Assim, o mtodo na classe base chamado apenas uma vez.

Quando Subclass.call_me executado, ele chama super().call_me, que pode ser uma das duas classes. Conforme a sada, foi chamado LeftSubclass.call_me. Este, por sua vez, ao executar super().call_me, ao invs de chamar o mtodo pai, chama o prximo mtodo, que RightSubclass.call_me. A sim, este chama o mtodo BaseClass.call_me. Portanto, as chamadas a super() garantem que cada mtodo da hierarquia seja chamado apenas uma vez.

Diferentes conjuntos de argumentos

Chamar super().call_me deu certo no ltimo exemplo pois os mtodos esperavam o mesmo nmero de parmetros. Mas os mtodos __init__ das superclasses de Friend esperam receber uma quantidade diferente de parmetros.

O problema de chamar mtodos de superclasse com nomes iguais, mas conjuntos diferentes de argumentos, surge ao realizar herana mltipla.

A soluo para esse problema aceitar keyword arguments (kwargs). kwargs uma forma de passar uma quantidade variada de argumentos ao mtodo que no foram especificados na declarao do mtodo. Assim, esse mtodo pode repassar os argumentos no utilizados ao prximo na chamada a super().

class Contact:
    all_contacts = []

    def __init__(self, name="", email="", **kwargs):
        super().__init__(**kwargs)
        self.name = name
        self.email = email
        self.all_contacts.append(self)


class AddressHolder:
    def __init__(self, street="", city="", state="", code="", **kwargs):
        super().__init__(**kwargs)
        self.street = street
        self.city = city
        self.state = state
        self.code = code


class Friend(Contact, AddressHolder):
    def __init__(self, phone="", **kwargs):
        super().__init__(**kwargs)
        self.phone = phone

A sintaxe **kwargs coleta argumentos no definidos na lista de parmetros do mtodo, armazenando-os em um dicionrio convencionalmente chamado kwargs. Ao chamar outros mtodos passando kwargs como argumento, este desempacotado, e os argumentos que coincidem com os parmetros do mtodo so removidos de kwargs. Abordaremos kwargs com mais detalhes no Captulo 7. Para saber mais a respeito, acesse aqui.

Embora tenhamos resolvido o problema, o cdigo no fica muito claro. Em casos onde no podemos fugir disso, docstrings podem ajudar a esclarecer quais argumentos so esperados pelo mtodo.

Ao passar a varivel **kwargs para super, a varivel no inclui nenhum dos outros parmetros. Isso pode causar problemas quando outras classes necessitarem desses parmetros. Para garantir que uma varivel seja transmitida para as classes superiores, existem diversas abordagens. Suponha que tanto a classe Contact quanto Friend precisem ser inicializadas com um parmetro phone. Algumas opes so:

  1. Deixar a varivel no dicionrio kwargs, em vez de inclu-la explicitamente como parmetro. As classes podem acess-la usando kwargs['phone'].
  2. Tornar a varivel um parmetro explcito, mas atualizar o dicionrio kwargs antes de pass-lo para super usando kwargs['phone'] = phone.
  3. Tornar a varivel um parmetro explcito, mas atualizar o dicionrio kwargs usando o mtodo kwargs.update, que til para adicionar vrios argumentos de uma vez.
  4. Passar a varivel explicitamente usando super().__init__(phone=phone, **kwargs).

kwargs um recurso poderoso do Python, mas lembre-se da boa prtica de programao: explcito melhor que implcito.

A herana mltipla bsica pode ser til, mas em muitos casos mais vantajoso optar por uma abordagem mais transparente, como a composio ou a aplicao de padres de design abordados nos captulos 10 e 11.

Polimorfismo

Definio: chamar um mtodo idntico para diferentes subclasses mas com comportamentos diferentes, sem explicitamente saber qual classe foi chamada. Por exemplo, um objeto da classe AudioFile pode ter um mtodo play que se encarrega de descomprimir o arquivo de udio e direcion-lo para a placa de som. Mas dependendo do tipo de udio, o processo de descompresso diferente. Assim, poderamos ter diferentes subclasses com o mesmo mtodo implementado de forma diferente para diferentes arquivos de udio, como mp3, wav, ogg. Um objeto tocador de msica, ento, no precisaria saber a subclasse de um objeto AudioFile, apenas que ele possui um mtodo play que cumprir seu papel quando chamado.

Exemplo:

class AudioFile:
    def __init__(self, filename):
        if not filename.endswith(self.ext):
            raise Exception("Invalid file format")

        self.filename = filename


class MP3File(AudioFile):
    ext = "mp3"

    def play(self):
        print(f"playing {self.filename} as mp3")


class WavFile(AudioFile):
    ext = "wav"

    def play(self):
        print(f"playing {self.filename} as wav")


class OggFile(AudioFile):
    ext = "ogg"

    def play(self):
        print(f"playing {self.filename} as ogg")

O mtodo __init__ da superclasse consegue acessar a varivel ext de diferentes subclasses. Polimorfismo nos permite isso.

Tente criar um script tocador de msica, instanciar alguns objetos, inclusive passando um formato de arquivo invlido para um deles.

Duck typing

Como em Python as verificaes de tipo geralmente so feitas em tempo de execuo ao invs de tempo de compilao, alm do polimorfismo hereditrio, tambm podemos usar o recurso duck typing do Python.

A expresso mais comum que ilustra o conceito de duck typing : "Se parece um pato, nada como um pato e grasna como um pato, ento provavelmente um pato". Isso significa que, em vez de verificar se um objeto de uma classe especfica, voc verifica se ele possui os mtodos e atributos esperados para executar uma determinada tarefa.

O duck typing nos permite criar objetos que funcionem de maneira semelhante aos tipos esperados, sem nos preocupar com herana ou implementao explcita de interfaces. Por exemplo:

class FlacFile:
    def __init__(self, filename):
        if not filename.endswith(".flac"):
            raise Exception("Invalid file format")

        self.filename = filename

    def play(self):
        print(f"playing {self.filename} as flac")

Apesar de FlacFile no herdar diretamente de AudioFile, podemos us-la como se fosse, pois implementa a mesma interface:

def play(objeto):
    objeto.play()


mp3file = MP3File("file.mp3")
flacfile = FlacFile("file.flac")

play(mp3file)
play(flacfile)

Mtodos de classe

Alm dos mtodos estticos e mtodos de instncia, o Python tambm tem os mtodos de classe. Os Mtodos de classe so mtodos que operam na classe em si, em vez de em instncias individuais. Eles so decorados com @classmethod e seu primeiro argumento se refere prpria classe em que o mtodo foi chamado, e convencionalmente o declaramos como cls.

Os Mtodos de classe tm sua aplicao em situaes mais complexas, como quando precisamos criar objetos de uma classe com diferentes configuraes de maneira flexvel, usando o padro de design chamado "fbrica." Alm disso, eles so utilizados em conceitos avanados de herana, que citaremos posteriormente. No entanto, em tarefas do dia a dia, geralmente no so essenciais e podem ser substitudos por alternativas mais simples, como mtodos estticos.

O seguinte cdigo um exemplo do padro factory, adaptado daqui:

class Pizza:
    def __init__(self, ingredientes):
        self.ingredientes = ingredientes

    @classmethod
    def marguerita(cls):
        return cls(["molho de tomate", "manjerico", "rodelas de tomate fresco", "azeitona", "queijo muarela", "organo salpicado"])

    @classmethod
    def calabresa(cls):
        return cls(["queijo", "molho de tomate", "calabresa em rodelas", "cebola", "tomate picado", "azeite", "organo"])


p1 = Pizza(["queijo", "azeitona preta", "ovo cozido", "presunto cozido", "cebola", "ervilha", "molho de tomate", "azeite"])
p2 = Pizza.marguerita()
p3 = Pizza.calabresa()

print("\nPizza de Portuguesa:", p1.ingredientes)
print("\nPizza de Marguerita:", p2.ingredientes)
print("\nPizza de Calabresa:", p3.ingredientes)

No mtodo de classe, em vez de receber uma instncia como o parmetro self, ele recebe a prpria classe como o parmetro cls. Isso nos permite chamar o construtor da classe que foi passada como parmetro. Voc pode estar se perguntando por que no chamamos diretamente o construtor da classe Pizza em vez de passar cls como parmetro. De fato, neste exemplo especfico, poderamos fazer isso e marcar o mtodo como @staticmethod. No entanto, em situaes em que a classe Pizza uma superclasse, podemos no querer chamar diretamente o construtor da Pizza, mas sim ter acesso classe filha, por exemplo, Calzone, que pode ter sido passada como parmetro. Portanto, de fato, mtodos de classe tm mais utilidade no contexto de herana.

Classes bases abstratas

Em Python, uma Classe Base Abstrata (ABC) define um conjunto de mtodos e propriedades que subclasses so obrigadas a implementar.

Usando uma classe base abstrata

Python organiza sua API usando classes abstratas. Um exemplo delas a classe Container, que possui o mtodo abstrato __contains__(self, x), que verifica se x est contido no objeto desta classe. Este mtodo implementado pelas classes list, str e dict. Poderamos criar um continer simplrio que verifica se um parmetro est contido no conjunto dos nmeros inteiros pares:

class OddContainer:
    def __contains__(self, x):
        return isinstance(x, int) and x % 2 == 0

A funo isinstance verifica se o primeiro argumento uma instncia do segundo argumento.

Mesmo no herdando de Container, OddContainer considerado pelo Python como Container:

>>> from collections import Container
>>> odd_container = OddContainer()
>>> isinstance(odd_container, Container)
True
>>> issubclass(OddContainer, Container)
True

A funo issubclass verifica se o primeiro argumento uma subclasse do segundo argumento.

Isso um exemplo de duck typing, podemos alcanar polimorfismo sem uso de herana.

Quando utilizamos o operador in no Python, ele chama implicitamente o mtodo __contains__ do operando direita passando o operando esquerda como parmetro:

>>> 1 in odd_container
True
>>> 2 in odd_container
False
>>> 3 in odd_container
True
>>> "a string" in odd_container
False

Criando uma classe abstrata

Uma das principais vantagens das classes abstratas a facilitao na criao de APIs. A classe abstrata estabelece um contrato, ou seja, define quais mtodos devem ser implementados pelas subclasses. Essa especificao da API essencial para que os desenvolvedores entendam como interagir com as classes baseadas na abstrao. Alm disso, a prpria classe abstrata, juntamente com comentrios e descries adequadas, compe uma parte valiosa da documentao da API. Isso permite que os desenvolvedores compreendam a estrutura da classe abstrata, seus mtodos e propriedades, alm de entenderem como criar novas classes que se encaixem nesse padro. Seguindo a API voc e outros desenvolvedores podem desenvolver diferentes verses de classes abstratas para diferentes propsitos, e outros desenvolvedores podero usar as diferentes verses de igual modo. Um exemplo comum de APIs em bancos de dados, tendo uma API em comum voc pode implement-la para um banco de dados especfico, e outras pessoas que utilizam dessa API podem trocar para o seu banco de dados simplesmente mudando para a tua implementao sem necessitar modificar o restante do sistema.

Vamos ilustrar ABC com o exemplo de mdia de udio:

import abc


class MediaLoader(abc.ABC):
    @abc.abstractmethod
    def play(self):
        pass

    @property
    @abc.abstractmethod
    def ext(self):
        pass


class Ogg(MediaLoader):
    ext = ".ogg"

    def play(self):
        pass


o = Ogg()
print(o)
print(issubclass(Ogg, MediaLoader))
print(isinstance(o, MediaLoader))


class Mp3:
    ext = ".mp3"

    def play(self):
        pass


m = Mp3()
print(m)
print(issubclass(Mp3, MediaLoader))
print(isinstance(m, MediaLoader))


class Wav(MediaLoader):
    pass


w = Wav()
print(w)
print(issubclass(Wav, MediaLoader))
print(isinstance(w, MediaLoader))

Para uma classe ser abstrata, deve herdar de ABC. Uma classe abstrata pode ter mtodos concretos. Para marcar um mtodo como abstrato, decore-o com @abc.abstractmethod. O que tiver essa decorao deve ser implementado na classe filha. J a decorao @property, que no tem a ver com ABC, marca um mtodo como propriedade, isto , ele considerado como uma varivel somente-leitura. Veremos mais sobre decoraes no captulo 10.

Observe que Ogg uma subclasse de MediaLoader, mas Mp3 no. Quando voc herda de uma classe base, o Python verifica automaticamente se a nova classe uma subclasse vlida da classe base. No entanto, em algumas situaes, voc pode querer personalizar ou relaxar essas verificaes de subclasse atravs de duck typing. Para tanto, acrescente esse mtodo MediaLoader:

    @classmethod
    def __subclasshook__(cls, Subclass):
    if cls is MediaLoader:
        if hasattr(Subclass, "play") and hasattr(Subclass, "ext"):
            return True

    return NotImplemented

O mtodo __subclasshook__ frequentemente usado em classes base abstratas (ABCs) e pode ser usado para controlar se uma determinada classe considerada uma subclasse da classe que define o mtodo __subclasshook__. Ele deve ser definido como um mtodo de classe (usando a decorao @classmethod) em uma classe base. O mtodo recebe dois argumentos: a classe sendo verificada e a classe potencialmente considerada uma subclasse. Esse mtodo chamado automaticamente durante a execuo das funes issubclass e isinstance, antes da verificao de herana.

Se o mtodo retornar True, o Python considerar a classe como uma subclasse vlida da classe base. Se o mtodo retornar False, o Python no considerar a classe como uma subclasse vlida da classe base. Se o mtodo retornar NotImplemented, o Python continuar a verificao de subclasse olhando para as superclasses.

O uso de __subclasshook__ mais avanado e geralmente no necessrio para a maioria das situaes. Ele mais til em cenrios onde voc deseja customizar a verificao de subclasse para casos especficos ou quando lida com estruturas de herana complexas.

Tipos Enum

Um Enum um conjunto de nomes simblicos vinculados a valores nicos. Eles so semelhantes a variveis globais, mas oferecem um mtodo repr mais til, agrupamento, segurana de tipo (type-safety) e alguns outros recursos.

Eles so teis quando voc precisa representar um conjunto fixo de constantes. Por exemplo, os dias da semana:

from enum import Enum
class Weekday(Enum):
    MONDAY = 1
    TUESDAY = 2
    WEDNESDAY = 3
    THURSDAY = 4
    FRIDAY = 5
    SATURDAY = 6
    SUNDAY = 7

Ou talvez as cores primrias RGB:

from enum import Enum
class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

Como voc pode ver, criar um Enum to simples quanto escrever uma classe que herda do prprio Enum.

Dependendo da natureza do enum, o valor de um membro pode ou no ser importante, mas de qualquer maneira esse valor pode ser usado para obter o membro correspondente:

>>> Weekday(3)
<Weekday.WEDNESDAY: 3>

Os membros do Enum tm um atributo que contm apenas o nome deles:

>>> print(Weekday.TUESDAY.name)
TUESDAY

Da mesma forma, eles tm um atributo para o valor deles:

>>> Weekday.WEDNESDAY.value
3

Se voc tiver um membro enum e precisar do nome ou valor dele:

>>> member = Color.RED
>>> member.name
'RED'
>>> member.value
1

Membros e atributos permitidos de Enums

Enums so classes Python e podem ter mtodos e atributos. Por exemplo, o seguinte Enum tem todos os planetas do nosso sistema solar, cada um com dois atributos associados:

from enum import Enum


class Planet(Enum):
    MERCURY = (3.303e23, 2.4397e6)
    VENUS = (4.869e24, 6.0518e6)
    EARTH = (5.976e24, 6.37814e6)
    MARS = (6.421e23, 3.3972e6)
    JUPITER = (1.9e27, 7.1492e7)
    SATURN = (5.688e26, 6.0268e7)
    URANUS = (8.686e25, 2.5559e7)
    NEPTUNE = (1.024e26, 2.4746e7)

    def __init__(self, mass, radius):
        self.mass = mass  # in kilograms
        self.radius = radius  # in meters

    def surface_gravity(self):
        # universal gravitational constant  (m3 kg-1 s-2)
        G = 6.67300e-11
        return G * self.mass / (self.radius * self.radius)

    def surface_weight(self, otherMass):
        return otherMass * self.surface_gravity()

Se o mtodo __init__ for definido, o valor do membro enum ser passado para ele.

O cdigo a seguir pega seu peso na Terra (em qualquer unidade) e imprime seu peso em todos os planetas (na mesma unidade):

earth_weight = float(input("What is your weight on Earth? "))
mass = earth_weight / Planet.EARTH.surface_gravity()
for p in Planet:
    print(f"Your weight on {p.name} is {p.surface_weight(mass)}")

A maioria dos casos usa inteiros para valores de enum. Usar inteiros curto e prtico (e fornecido por padro pela API Funcional), mas no estritamente forado. Na grande maioria dos casos de uso, no importa o valor de um enum. Mas se o valor for importante, os enums podem ter valores arbitrrios:

class Mood(Enum):
    FUNKY = 1
    HAPPY = 3

    def describe(self):
        # self is the member here
        return self.name, self.value

    def __str__(self):
        return f'my custom str! {self.value}'

    @classmethod
    def favorite_mood(cls):
        # cls here is the enumeration
        return cls.HAPPY

Ento:

>>> Mood.favorite_mood()
<Mood.HAPPY: 3>
>>> Mood.HAPPY.describe()
('HAPPY', 3)
>>> str(Mood.FUNKY)
'my custom str! 1'

Como outro exemplo, datetime.date tem dois mtodos para retornar o dia da semana: weekday e isoweekday. A diferena que um deles conta de 0-6 e o outro de 1-7. Em vez de nos preocuparmos com isso manualmente, podemos adicionar um mtodo ao enum Weekday para extrair o dia da instncia de data e retornar o membro enum correspondente:

class Weekday(Enum):
    MONDAY = 1
    TUESDAY = 2
    WEDNESDAY = 3
    THURSDAY = 4
    FRIDAY = 5
    SATURDAY = 6
    SUNDAY = 7

    @classmethod
    def from_date(cls, date):
        return cls(date.isoweekday())

Agora podemos descobrir que dia hoje! Observe:

>>> from datetime import date
>>> Weekday.from_date(date.today())
<Weekday.TUESDAY: 2>

Este enum Weekday timo se nossa varivel precisar apenas de um dia, mas e se precisarmos de vrios? Talvez estejamos escrevendo uma funo para planejar tarefas durante uma semana e no queremos usar uma lista...

Flag Enum

Ao criar um enum que herda de Flag, podemos executar operaes bit a bit em seus membros, como combin-los usando o operador |:

from enum import Flag

class Weekday(Flag):
    MONDAY = 1
    TUESDAY = 2
    WEDNESDAY = 4
    THURSDAY = 8
    FRIDAY = 16
    SATURDAY = 32
    SUNDAY = 64

Mudamos duas coisas: herdamos de Flag, e os valores so todos potncias de 2.

Assim como o enum original Weekday acima, podemos ter uma nica seleo:

>>> first_week_day = Weekday.MONDAY
>>> first_week_day
<Weekday.MONDAY: 1>

Mas o Flag tambm nos permite combinar vrios membros em uma nica varivel:

>>> weekend = Weekday.SATURDAY | Weekday.SUNDAY
>>> weekend
<Weekday.SATURDAY|SUNDAY: 96>

Nos casos em que os valores reais dos membros no importam, voc pode economizar algum trabalho e usar o mtodo auto para os valores:

from enum import auto
class Weekday(Flag):
    MONDAY = auto()
    TUESDAY = auto()
    WEDNESDAY = auto()
    THURSDAY = auto()
    FRIDAY = auto()
    SATURDAY = auto()
    SUNDAY = auto()
    WEEKEND = SATURDAY | SUNDAY

Considere este cdigo:

from enum import Flag
from enum import auto


class Color(Flag):
    BLACK = 0
    RED = auto()
    GREEN = auto()
    BLUE = auto()
    PURPLE = RED | BLUE
    WHITE = RED | GREEN | BLUE

Iterar sobre uma combinao de membros Flag retornar apenas os membros que so compostos por um nico bit:

>>> Color(5)  # named combination
<Color.PURPLE: 5>
>>> Color(3)      # not named combination
<Color.GREEN|RED: 3>

Nomes de pseudo-flags so construdos a partir dos nomes de seus membros:

>>> (Color.RED | Color.BLUE).name
'PURPLE'

Flags de valor zero so sempre consideradas contidas:

>>> Color.BLACK in Color.WHITE
True

Caso contrrio, ser retornado True apenas se todos os bits de uma flag estiverem na outra flag:

>>> Color.PURPLE in Color.WHITE
True
>>> Color.GREEN in Color.PURPLE
False

    Descrio:

    Um tringulo um polgono com trs lados. Sejam $a$, $b$ e $c$ as medidas dos lados do tringulo. Seu semipermetro dado pela frmula $p=\frac{a+b+c}{2}$. Sua rea dada pela frmula $A=\sqrt{p(p-a)(p-b)(p-c)}$.

    Um quadriltero um polgono com quatro lados.

    Um quadrado um quadriltero com quatro lados de igual medida e todos os ngulos medindo 90.

    Um trapzio um quadriltero com apenas duas linhas paralelas entre si, chamadas de base maior e base menor. Sua altura a distncia perpendicular entre suas duas bases, dada pela frmula $$ h = \frac{ \sqrt{(-a + b + c + d)(a - b + c + d)(a - b - c + d)(a - b + c - d)} } {2\vert a - b \vert}$$, onde $a$, $b$, $c$ e $d$ representam as medidas das duas bases e dos dois lados oblquos, respectivamente. Sua rea dada pela frmula $A=\frac{a+b}{2}\cdot h$.
