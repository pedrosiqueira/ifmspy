Objetos em Python

Criando classes em Python

O Python procura ser menos boilerplate, como é demonstrado pelo 'hello world' em apenas uma linha de código.

# first_class.py
class MyFirstClass:
    pass

Palavra-chave class, nome da classe e dois pontos. Palavra-chave pass para indicar um bloco vazio de código, já que o Python utiliza espaços e não chaves para delimitar blocos de código. O PEP 8 recomenda o uso de PascalCase para o nome das classes e 4 espaços para indentação. Para executar o código e depois entrar no interpretador interativo, execute python -i first_class.py.

Os endereços de memória demonstram que existem dois objetos distintos envolvidos.

Adicionando atributos

Podemos definir atributos arbitrários em um objeto instanciado usando a notação de ponto:

# point.py
class Point:
    pass

p1 = Point()
p2 = Point()
p1.x = 5
p1.y = 4
p2.x = 3
p2.y = 6
print(p1.x, p1.y)
print(p2.x, p2.y)

Adicionando um método

Vamos criar um método:

class Point:
    def reset(self):
        self.x = 0
        self.y = 0

p = Point()
p.reset()
print(p.x, p.y)

O argumento self de um método é uma referência ao objeto no qual o método está sendo invocado. Esse argumento é convencionalmente chamado de self.

Observe que quando chamamos o método p.reset(), não precisamos passar o argumento self para ele. O Python cuida disso automaticamente para nós. Ele sabe que estamos chamando um método no objeto p e automaticamente passa esse objeto para o método.

No entanto, o método é apenas uma função que está em uma classe. Em vez de chamar o método no objeto, poderíamos invocar a função na classe, passando explicitamente nosso objeto como argumento self:

>>> p = Point()
>>> Point.reset(p)
>>> print(p.x, p.y)

O que acontece se esquecermos de incluir o argumento self em nossa definição de classe? O Python apresentará uma mensagem de erro.

Mais argumentos

Vamos adicionar um novo método que nos permite mover um ponto para uma posição arbitrária, não apenas para a origem. Também podemos incluir um método que aceita outro objeto Point como entrada e retorna a distância entre eles:

class Point:
    def move(self, x, y):
        self.x = x
        self.y = y

    def reset(self):
        self.move(0, 0)

    def calculate_distance(self, other_point):
        return ((self.x - other_point.x) ** 2 + (self.y - other_point.y) ** 2) ** (1/2)


point1 = Point()
point2 = Point()

point1.reset()
point2.move(5, 0)
print(point2.calculate_distance(point1))
assert point2.calculate_distance(point1) == point1.calculate_distance(point2)
point1.move(3, 4)
print(point1.calculate_distance(point2))
print(point1.calculate_distance(point1))

A função assert é uma ferramenta simples de teste; o programa apresentará um erro se a afirmação após o assert for avaliada como Falso. Neste caso, usamos para garantir que a distância seja a mesma, independentemente de qual ponto chamou o método calculate_distance do outro ponto.

Inicializando o objeto

Se não definirmos explicitamente as posições x e y em nosso objeto Point, seja usando o método move ou acessando-as diretamente, teremos um ponto com defeito e sem posição real. O que acontecerá quando tentarmos acessá-lo?

Todas as mensagens de erro seguem esse padrão.

Poderíamos definir valores padrão para esses atributos. A maioria das linguagens de programação orientada a objetos possui o conceito de um construtor, um método especial que cria e inicializa o objeto quando ele é criado. No Python, existem dois tipos de métodos para construir objetos: __new__ e __init__. A menos que você esteja fazendo algo muito exótico, usará o método de inicialização muito mais comum, __init__.

Métodos e variáveis ‘mágicas’ no Python começam e terminam com duplos sublinhados. Não identifique os seus próprios assim! Você também não deve acessar diretamente variáveis ou métodos mágicos, eles são utilizados automaticamente pelo Python quando necessário.

class Point:
    def __init__(self, x, y):
        self.move(x, y)

    def move(self, x, y):
        self.x = x
        self.y = y

    def reset(self):
        self.move(0, 0)

# Construindo um ponto
point = Point(3, 5)
print(point.x, point.y)

Agora só podemos construir um ponto passando os argumentos x e y.

Se não quisermos tornar os dois argumentos obrigatórios, podemos usar a mesma sintaxe que as funções Python usam para fornecer argumentos padrão.

class Point:
    def __init__(self, x=0, y=0):
        self.move(x, y)

Explicando-se

O Python suporta documentar sua API por meio do uso de docstrings. Cada classe, função ou cabeçalho de método pode ter uma string Python padrão como a primeira linha após a definição (a linha que termina com dois pontos). Para docstrings de uma linha, use apóstrofos (') ou aspas ("). Para docstrings de várias linhas, use três apóstrofos (''') ou três aspas (""").

Uma docstring deve resumir clara e concisamente o objetivo da classe ou método que está descrevendo. Deve explicar quaisquer parâmetros cujo uso não seja imediatamente óbvio e também é um bom lugar para incluir exemplos curtos de como usar a API. Quaisquer avisos ou problemas que um usuário desavisado da API deva estar ciente também devem ser observados.

class Point:
    "Representa um ponto em coordenadas geométricas bidimensionais."

    def __init__(self, x=0, y=0):
        """Inicializa a posição de um novo ponto. As coordenadas x e y
           podem ser especificadas. Se não forem, o
           ponto assume a origem."""
        self.move(x, y)

    def move(self, x, y):
        "Move o ponto para uma nova localização no espaço 2D."
        self.x = x
        self.y = y

    def reset(self):
        "Redefine o ponto para a origem geométrica: 0, 0."
        self.move(0, 0)

    def calculate_distance(self, other_point):
        """Calcula a distância deste ponto para um segundo
        ponto passado como parâmetro.

        Esta função usa o Teorema de Pitágoras para calcular
        a distância entre os dois pontos. A distância é
        retornada como um número float."""

        return (
            (self.x - other_point.x) ** 2
            + (self.y - other_point.y) ** 2
        ) ** (1/2)


# como usar:
point1 = Point()
point2 = Point()

point1.reset()
point2.move(5, 0)
print(point2.calculate_distance(point1))
assert point2.calculate_distance(point1) == point1.calculate_distance(point2)
point1.move(3, 4)
print(point1.calculate_distance(point2))
print(point1.calculate_distance(point1))

Módulos e pacotes

Em programas pequenos, colocamos nossas classes e funções em um arquivo e um pouco de script no final do arquivo. Conforme nosso projeto cresce, organizamos o código em módulos, que são simplesmente arquivos Python. Se um módulo precisa de um recurso de outro módulo, ele deve importar esse recurso. Um recurso pode ser uma classe, método ou variável.

Por exemplo, em um sistema de e-commerce, poderíamos ter o módulo database.py contendo classes e funções relacionadas ao banco de dados. Outro módulo, chamado products.py, responsável por operações com produtos, poderia usar a classe Database de database.py através de uma das seguintes alternativas:

  • import database
db = database.Database()
# Do queries on db
            
  • from database import Database
db = Database()
# Do queries on db
            
  • from database import Database as DB
db = DB()
# Do queries on db
            
  • from database import Database, Query
db = Database()
q = Query()
# Do queries on db with q

Essas variações permitem que importemos: o módulo inteiro; recurso(s) específico(s); ou recurso(s) renomeado(s) para evitar conflitos de namespace.

Você pode importar todos os recursos de um módulo com from database import *, mas isso não é recomendado. Isso dificulta a identificação de onde um recurso veio, aumenta a possibilidade de conflitos de nomes com recursos que você nem vai utilizar e, pior, também importa as importações indiretas do módulo importado.

Organizando módulos

Conforme um projeto cresce, é útil organizar módulos em pacotes, que são pastas que agregam vários módulos, ou até mesmo outros pacotes, dentro delas. Para que uma pasta seja considerada um pacote, você deve colocar um arquivo __init__.py dentro dela, caso contrário, não será possível importar módulos dessa pasta.

Suponha a seguinte estrutura de pastas:

root/
    main.py
    ecommerce/
        __init__.py
        database.py
        products.py
        payments/
            __init__.py
            square.py
            stripe.py

Existem dois modos de importação: importação absoluta e importação relativa.

Importações absolutas

Importações absolutas especificam o caminho completo para o recurso que está sendo importado. Por exemplo, para importar a classe Product do módulo products no pacote ecommerce, poderíamos usar qualquer uma das seguintes sintaxes:

  • import ecommerce.products
produto = ecommerce.products.Product()
  • from ecommerce import products
produto = products.Product()
  • from ecommerce.products import Product
produto = Product()

Essas instruções funcionarão de qualquer módulo. Entretanto, especialmente ao importar de uma pasta pai, você precisa executar o Python da pasta raiz do projeto com a opção -m. Por exemplo, supondo que você esteja na pasta root, para executar square.py:

python3 -m ecommerce.payments.square

Você também pode carregar pacotes da pasta de pacotes instalados do Python ou da variável de ambiente PYTHONPATH.

A escolha da sintaxe depende do gosto pessoal e das necessidades específicas do aplicativo. Se muitas classes ou funções de um módulo forem necessárias, pode ser útil importar o módulo inteiro. Se apenas algumas classes ou funções forem necessárias, elas podem ser importadas diretamente.

Importações relativas

Você também pode importar recursos a partir do caminho atual, parecido com a navegação de pastas pelo terminal. Basta começar a importação com um ponto, que representa o pacote atual. Por exemplo, para importar Database de dentro de products.py:

from .database import Database

Cada ponto adicional acessa a próxima pasta pai. Por exemplo, para importar Database de dentro de square.py:

from ..database import Database

Também é possível importar código diretamente de pacotes usando o arquivo __init__.py. Esse arquivo pode conter recursos disponíveis como parte do pacote. Por exemplo, se o arquivo ecommerce/__init__.py contiver a linha from .database import db, poderíamos acessar o atributo db usando a seguinte importação: from ecommerce import db.

No entanto, geralmente é recomendado evitar colocar muito código no arquivo __init__.py, pois isso pode dificultar a localização de onde o código é declarado e pode levar a comportamentos inesperados.

Organizando o conteúdo do módulo

Dentro de um módulo, podemos especificar variáveis, classes ou funções. Essas podem ser usadas para armazenar estados globais sem conflitos de namespace. Por exemplo, poderíamos criar um único objeto de banco de dados disponível globalmente no módulo de banco de dados, definindo uma classe Database e instanciando-a no nível do módulo:

class Database:
    # implementação do banco de dados
    pass

database = Database()

Esse objeto pode então ser importado e usado em outros módulos com from ecommerce.database import database. No entanto, essa abordagem pode ter desvantagens, pois o objeto do banco de dados é criado imediatamente quando o módulo é importado, o que pode nem sempre ser desejável (inicialização lenta ou informações indisponíveis). Para adiar a criação do banco de dados até que ele seja realmente necessário, poderíamos definir uma função initialize_database:

class Database:
    # implementação do banco de dados
    pass

database = None

def initialize_database():
    global database
    database = Database()

A palavra-chave global é usada para especificar que a variável database dentro da função se refere à variável de nível de módulo. Isso nos permite adiar a criação do banco de dados até que ele seja realmente necessário.

Todo o código de nível de módulo é executado imediatamente quando o módulo é importado. No entanto, o código dentro de uma função ou método não é executado até que a função seja chamada. Isso pode ser útil para organizar o código de inicialização e evitar a execução não intencional ao importar módulos.

Para evitar a execução do código de inicialização ao importar um módulo, recomenda-se colocar o código de inicialização em uma função (geralmente chamada main) e executar essa função apenas se o módulo está sendo executado como um script:

class UsefulClass:
    """Esta classe pode ser útil para outros módulos."""
    pass

def main():
    """Cria uma classe útil e faz algo com ela para o nosso módulo."""
    util = UsefulClass()
    print(util)

if __name__ == "__main__":
    main()

A variável especial __name__ especifica o nome do módulo quando ele é importado. Quando o módulo é executado como um script, seu valor é definido como "__main__".

Tipicamente, métodos vão em classes, que vão em módulos, que vão em pacotes. Mas em Python, recursos podem ser definidos em qualquer lugar, inclusive dentro de funções ou métodos. Isso pode ser útil para criar classes temporárias que são necessárias apenas em um escopo específico:

def format_string(string, formatter=None):
    """Formata uma string usando o objeto formatter, que
    espera ter um método format() que aceite uma string."""

    class DefaultFormatter:
        """Formata uma string em maiúsculas."""

        def format(self, string):
            return str(string).title()

    if not formatter:
        formatter = DefaultFormatter()

    return formatter.format(string)

hello_string = "hello world, how are you today?"
print(" input: " + hello_string)
print("output: " + format_string(hello_string))

No entanto, essa técnica não é comumente usada em Python.

Quem pode acessar meus dados?

O Python não tem um controle de acesso estrito como algumas outras linguagens de programação orientadas a objetos (public, package, protected, private). Em vez disso, ele depende de convenções e boas práticas para indicar quais métodos e atributos devem ser considerados privados (ou internos). Por convenção, recursos internos são prefixados com um caractere de sublinhado (_). Ou você pode escrever docstrings claras indicando isso. No entanto, não há nada no interpretador que impeça o acesso direto a eles.

Você também pode prefixar identificadores com dois sublinhados. O Python irá realizar o name mangling, que é a mudança de nome desses identificadores, tornando mais difícil acessá-los:

class SecretString:
    """Uma maneira não muito segura de armazenar uma string secreta."""

    def __init__(self, string_normal, senha):
        self.__string_normal = string_normal
        self.__frase_senha = senha

    def descriptografar(self, senha):
        """Somente mostra a string se a senha estiver correta."""
        if senha == self.__senha:
            return self.__string_normal
        else:
            return ""

Quando usamos um duplo sublinhado, o recurso é prefixado com _<nomedaclasse>. Quando os métodos dentro da classe acessam a variável internamente, ela é automaticamente 'unmangled'. Quando classes externas desejam acessá-la, elas precisam fazer o name mangling manualmente. Portanto, o name mangling não garante privacidade; ele apenas recomenda fortemente.

Bibliotecas de terceiros

O Python inclui uma extensa biblioteca padrão. Mas o forte da linguagem é a gama de bibliotecas de terceiros que você pode utilizar

“Um bom programador é um programador preguiçoso...”, uma maneira diferente de dizer “Não reinvente a roda!”. Você pode encontrar as bibliotecas necessárias no Índice de Pacotes Python (PyPI) em http://pypi.python.org/. A ferramenta 'pip' permite a fácil instalação desses pacotes.

Nota: O Python 3.4+ contém 'ensurepip' para instalar o 'pip'. Em sistemas Unix, use 'sudo' antes do comando.

python -m ensurepip

Uma vez que o 'pip' está instalado e você sabe o nome do pacote, você pode instalá-lo usando a seguinte sintaxe:

pip install package_name

No entanto, não é comum instalar uma biblioteca de terceiros diretamente no diretório Python do sistema. O Python 3.4+ oferece a ferramenta 'venv' para criar um ambiente virtual com sua própria instalação Python. Ativar o ambiente garante que os comandos Python afetem apenas o diretório de trabalho. Para usá-lo:

cd project_directory
    python -m venv .env # criar um venv
    source .env/bin/activate # ativar venv no Linux ou macOS
    .env/bin/activate.bat # ativar venv no Windows

Cada projeto Python pode ter seu próprio ambiente virtual. Armazenar ambientes no diretório do projeto, enquanto os ignora no controle de versão, é uma abordagem comum.

Os ambientes virtuais isolam dependências de terceiros, úteis quando os projetos requerem diferentes versões de bibliotecas. Conflitos entre pacotes instalados pelo sistema e pacotes instalados pelo pip são evitados, promovendo um gerenciamento eficiente de pacotes.

Estudo de caso

Para ilustrar os conceitos, vamos criar um aplicativo simples de bloco de notas para linha de comando.

Começaremos com uma análise básica: o aplicativo gerenciará pequenas anotações chamadas notas, armazenadas em um bloco de notas. Cada nota inclui a data de criação e tags (etiquetas) opcionais para fácil categorização. Os usuários devem ser capazes de editar, pesquisar e modificar as notas.

Os objetos-chave incluem o objeto Note para notas individuais e um objeto contêiner NoteBook. Embora datas e etiquetas pudessem ser tratadas como objetos, para este protótipo, podemos usar a biblioteca padrão do Python para datas e usar uma string separada por vírgulas para etiquetas.

Os objetos Note possuem atributos memo, tags e creation_date. Cada nota também requer um id para seleção do usuário em um menu. Para facilitar a pesquisa, um método match no objeto Note pode determinar se uma nota corresponde a uma determinada string, centralizando ajustes relacionados à pesquisa.

O objeto NoteBook contém uma lista de notas como atributo e requer um método search para pesquisas filtradas.

A interação com esses objetos ocorre através de um menu de linha de comando. Podemos projetar a interface já pensando em acomodar futuras interfaces como GUI ou web.

A classe NoteBook deve incluir métodos para essas interfaces. Precisaremos de funções para adicionar novas notas, modificar notas existentes por ID e realizar pesquisas. Listar todas as notas pode ser alcançado acessando diretamente o atributo da lista de notas.

A implementação da classe Note pode ficar assim:

import datetime

# Store the next available id for all new notes
last_id = 0


class Note:
    """Represent a note in the notebook. Match against a
    string in searches and store tags for each note."""

    def __init__(self, memo, tags=""):
        """initialize a note with memo and optional
        space-separated tags. Automatically set the note's
        creation date and a unique id."""
        self.memo = memo
        self.tags = tags
        self.creation_date = datetime.date.today()
        global last_id
        last_id += 1
        self.id = last_id

    def match(self, filter):
        """Determine if this note matches the filter
        text. Return True if it matches, False otherwise.

        Search is case sensitive and matches both text and
        tags."""
        return filter in self.memo or filter in self.tags

A implementação da classe NoteBook pode ficar assim:

class Notebook:
    """Represent a collection of notes that can be tagged,
    modified, and searched."""

    def __init__(self):
        """Initialize a notebook with an empty list."""
        self.notes = []

    def new_note(self, memo, tags=""):
        """Create a new note and add it to the list."""
        self.notes.append(Note(memo, tags))

    def modify_memo(self, note_id, memo):
        """Find the note with the given id and change its
        memo to the given value."""
        for note in self.notes:
            if note.id == note_id:
                note.memo = memo
                break

    def modify_tags(self, note_id, tags):
        """Find the note with the given id and change its
        tags to the given value."""
        for note in self.notes:
            if note.id == note_id:
                note.tags = tags
                break

    def search(self, filter):
        """Find all notes that match the given filter
        string."""
        return [note for note in self.notes if note.match(filter)]

Métodos e variáveis, de instância e estáticos

Métodos e variáveis de instância:

Instâncias são sinônimos de objetos. Métodos de instância são aqueles que operam em instâncias da classe e têm acesso aos atributos específicos de uma única instância. Eles são definidos com o primeiro argumento, convencionalmente declarado como self, se referindo à própria instância. Métodos de instância são sinônimos de métodos. Atributos são sinônimos de variáveis de instância ou de campos. Alguns ainda consideram atributos sinônimos de propriedades, mas este termo é mais utilizado no sentido de atributos “somente-leitura”.

class Dog:
        def __init__(self, name):
            self.name = name

        def bark(self):
            print(f"{self.name} is barking!")

    dog1 = Dog("Buddy")
    dog2 = Dog("Charlie")

    dog1.bark()  # Output: Buddy is barking!
    dog2.bark()  # Output: Charlie is barking!

Nesse exemplo, bark é um método de instância que opera em instâncias específicas de Dog.

Métodos estáticos e variáveis de classe

Variáveis de classe agem como se fossem variável globais, só que estão associadas à classe. Assim como cada instância tem sua própria variável de instância com seu próprio valor, cada classe tem sua própria variável de classe, com seu próprio valor.

Semelhantemente, métodos estáticos agem como se fossem funções. Em vez de serem declarados no nível de módulo, são declarados no nível de classe. Eles podem ser decorados com @staticmethod e não recebem argumentos especiais como self. Em algumas linguagens de programação, métodos estáticos também são chamados de métodos de classe, mas no Python, métodos de classe têm a ver com outra coisa, que veremos mais para frente.

Você opta por variáveis de classe em vez de variáveis globais quando essas variáveis estão relacionadas apenas à classe em que foram declaradas. São um meio de organizar seu código encapsulando dados dentro da classe, de modo que para serem acessíveis, você deve especificar o nome da classe. Elas são frequentemente utilizadas para armazenar constantes, dados ou configurações relacionadas exclusivamente à classe ou dados compartilhados entre instâncias desta classe.

Por outro lado, as variáveis globais estão relacionadas ao módulo como um todo, o que inclui todas as classes que possam ter sido declaradas dentro desse módulo. Geralmente, variáveis globais são usadas para definir constantes ou configurações que são aplicáveis a diversas classes ou funções em um módulo, ou para armazenar dados que precisam ser acessados globalmente. Entretanto, é recomendável limitar o escopo de suas variáveis, evitando o uso excessivo de variáveis globais, o que pode tornar seu código menos organizado.

Da mesma forma, os métodos estáticos são a alternativa às funções quando se trata de realizar ações específicas relacionadas apenas a uma classe. Eles são úteis para funções utilitárias associadas a uma classe, mas que não requerem acesso a dados específicos de instâncias, operações que não dependem do estado de uma instância e funções que se encaixam logicamente na estrutura da classe.

Já funções regulares são independentes de classes. São ideais como funções utilitárias de uso geral, não diretamente vinculadas a uma classe específica, funções que executam operações envolvendo dados de diversas fontes ou funções usadas em diferentes módulos ou classes.

Em resumo, use métodos estáticos quando tiver funcionalidade intimamente associada a uma classe e use funções regulares quando precisar de funcionalidade independente, reutilizável e acessível globalmente. A escolha depende do contexto e do design do seu código.

class Date(object):
    num_instances = 0

    def __init__(self, year=0, month=0, day=0):
        self.day = day
        self.month = month
        self.year = year
        Date.num_instances += 1

    def is_leap_year(self):
        return self.year % 4 == 0 and self.year % 100 != 0 or self.year % 400 == 0

    @staticmethod
    def from_string(date_as_string):
        list = [int(x) for x in date_as_string.split("-")]
        date = Date(list[0], list[1], list[2])
        return date

    def is_date_valid(date_as_string):
        list = [int(x) for x in date_as_string.split("-")]
        return list[0] <= 3999 and list[1] <= 12 and list[2] <= 31


date1 = Date(2100, 12, 25)
print(date1.day, date1.month, date1.year, date1.is_leap_year())
print("Number of instances:", Date.num_instances)
date_string = "2024-02-29"
if Date.is_date_valid(date_string):
    date2 = Date.from_string(date_string)
    print(date2.day, date2.month, date2.year, date2.is_leap_year())
print("Number of instances:", Date.num_instances)

Variáveis de classe vs variáveis de instância vs variáveis globais

Você pode importar e utilizar variáveis globais e funções diretamente de outros módulos. Mas para utilizar variáveis de classe e métodos estáticos, você deve especificar o nome da classe antes. Isso lhe dá um maior controle sobre o escopo dessas variáveis e métodos, garantindo uma organização mais eficaz do seu código.

Considere o módulo test com o seguinte código:

global_var = "a global variable"


def func(par):  # Function
    print(global_var, "and", par)


class Test:
    class_var = "a class variable"  # Class variable

    def __init__(self, par):
        self.inst_var = par  # Instance variable

    def test1(self, par):  # Instance method
        print(self.inst_var, "and", par)

    @staticmethod
    def test3(par):  # Static method
        print(Test.class_var, "and", par)

    def test4(par):  # Static method
        print(Test.class_var, "and", par)

A única diferença entre usar ou não a decoração @staticmethod, é que sem ela você não pode chamar um método estático através de uma instância. Como não é recomendável chamar métodos estáticos e métodos de classe através de instâncias, então essa decoração pode ser omitida.

Considere o módulo main com o seguinte código:

from test import Test
# from test.Test import test2 # error!
from test import func
from test import global_var


def invoking_methods():
    func("a argument")

    t = Test("a instance variable")
    t.test1("a argument")
    t.test2("a argument")

    Test.test2("other argument")
    Test.test3("other argument")

    # t.test3("cannot do this")  # error!
    # Test.test1("this is also forbidden")  # error!

A função invoking_methods chama cada um dos métodos através de uma instância e através de uma classe.

Observe as linhas 2, 17 e 18 comentadas. Não é possível importar métodos estáticos nem variáveis de classe diretamente, não é possível chamar um método estático não decorado através de uma instância, e não é possível chamar um método de instância através de uma classe.

Considere o seguinte trecho de código adicionado ao módulo main:

def accessing_variables():
    t1 = Test("attribute 1")
    t2 = Test("attribute 2")

    print(t1.inst_var, "|", t1.class_var)
    print(t2.inst_var, "|", t2.class_var)
    print(Test.class_var, "|", global_var)

    t1.inst_var = "attribute 1 modified"
    t1.class_var = "new attribute"
    Test.class_var = "class var modified"
    # global_var = "a local var" # error!

    print(t1.inst_var, "|", t1.class_var)
    print(t2.inst_var, "|", t2.class_var)
    print(Test.class_var, "|", global_var)

    # print(Test.inst_var) # error!

A função accessing_variables acessa variáveis de instância e de classe por meio de instâncias e da classe, além de acessar uma variável global.

Observe a linha 12 comentada. Se uma função ou método não possuir uma variável local com esse nome, ela procura acessar a variável global. Mas não é possível modificar variáveis globais sem antes especificá-las como globais usando a palavra-chave global. Se tentar fazer isso, na verdade, o Python interpretará como uma tentativa de criar uma nova variável local. Portanto, não é recomendável acessar variáveis globais sem antes especificá-las como globais.

Agora observe a linha 18 comentada. Uma classe não consegue acessar variáveis de instâncias. Já uma instância consegue acessar as variáveis de sua classe como se fossem seus atributos. Se uma instância não possuir um atributo com esse nome, ela procura acessar a variável de classe. Mas não é possível modificar variáveis de classe através da instância. Se tentar fazer isso, na verdade, estará criando um novo atributo. Portanto, não é recomendável acessar variáveis de classe através de instâncias.

O que são objetos?

Em Python, tudo é objeto. Mas o que é um objeto? Linguagens de programação diferentes definem "objeto" de maneiras diversas. Em algumas, objetos são aqueles que possuem atributos e métodos; em outras, são aqueles que podem ser herdados.

No Python, a definição é mais flexível. Alguns objetos não têm atributos ou métodos, e nem todos os objetos podem ser herdados. No entanto, tudo é objeto no sentido de que pode ser atribuído a uma variável ou passado como argumento para uma função. Portanto, funções, classes e números são objetos! Por exemplo, todas as funções têm o atributo doc, que retorna a doc-string definida no código-fonte da função.

No Python, todo objeto possui o atributo __dict__, que é um dicionário contendo os atributos do objeto e seus valores:

class Test:
    """uma classe de teste"""

    class_var = 10

    def test(self):
        self.inst_var = 20


def func(par):
    """uma função que imprime os atributos `__doc__` e `__dict__` do parâmetro `par`"""
    print("\n", par.__doc__, par.__dict__)


t = Test()
t.test()
func(t)
func(Test)
func(func)
