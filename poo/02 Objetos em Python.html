Criando classes em Python

Palavra-chave class, nome da classe e dois pontos. Palavra-chave pass para indicar um bloco vazio de cdigo, j que o Python utiliza espaos e no chaves para delimitar blocos de cdigo. Adicionando um mtodo

Vamos criar um mtodo:

class Point:
    def reset(self):
        self.x = 0
        self.y = 0

p = Point()
p.reset()
print(p.x, p.y)

O argumento self de um mtodo uma referncia ao objeto no qual o mtodo est sendo invocado. Esse argumento convencionalmente chamado de self.

Observe que quando chamamos o mtodo p.reset(), no precisamos passar o argumento self para ele. O Python cuida disso automaticamente para ns. Ele sabe que estamos chamando um mtodo no objeto p e automaticamente passa esse objeto para o mtodo.

No entanto, o mtodo apenas uma funo que est em uma classe. Em vez de chamar o mtodo no objeto, poderamos invocar a funo na classe, passando explicitamente nosso objeto como argumento self:

>>> p = Point()
>>> Point.reset(p)
>>> print(p.x, p.y)

O que acontece se esquecermos de incluir o argumento self em nossa definio de classe? O Python apresentar uma mensagem de erro.

Mais argumentos

Vamos adicionar um novo mtodo que nos permite mover um ponto para uma posio arbitrria, no apenas para a origem. Tambm podemos incluir um mtodo que aceita outro objeto Point como entrada e retorna a distncia entre eles:

class Point:
    def move(self, x, y):
        self.x = x
        self.y = y

    def reset(self):
        self.move(0, 0)

    def calculate_distance(self, other_point):
        return ((self.x - other_point.x) ** 2 + (self.y - other_point.y) ** 2) ** (1/2)


point1 = Point()
point2 = Point()

point1.reset()
point2.move(5, 0)
print(point2.calculate_distance(point1))
assert point2.calculate_distance(point1) == point1.calculate_distance(point2)
point1.move(3, 4)
print(point1.calculate_distance(point2))
print(point1.calculate_distance(point1))

A funo assert uma ferramenta simples de teste; o programa apresentar um erro se a afirmao aps o assert for avaliada como Falso. Neste caso, usamos para garantir que a distncia seja a mesma, independentemente de qual ponto chamou o mtodo calculate_distance do outro ponto.

Inicializando o objeto

Se no definirmos explicitamente as posies x e y em nosso objeto Point, seja usando o mtodo move ou acessando-as diretamente, teremos um ponto com defeito e sem posio real. O que acontecer quando tentarmos acess-lo?

Todas as mensagens de erro seguem esse padro.

Poderamos definir valores padro para esses atributos. A maioria das linguagens de programao orientada a objetos possui o conceito de um construtor, um mtodo especial que cria e inicializa o objeto quando ele criado. No Python, existem dois tipos de mtodos para construir objetos: __new__ e __init__. A menos que voc esteja fazendo algo muito extico, usar o mtodo de inicializao muito mais comum, __init__.

Mtodos e variveis mgicas no Python comeam e terminam com duplos sublinhados. No identifique os seus prprios assim!

Agora s podemos construir um ponto passando os argumentos x e y.

Explicando-se

O Python suporta documentar sua API por meio do uso de docstrings. Cada classe, funo ou cabealho de mtodo pode ter uma string Python padro como a primeira linha aps a definio (a linha que termina com dois pontos). Para docstrings de uma linha, use apstrofos (') ou aspas ("). Para docstrings de vrias linhas, use trs apstrofos (''') ou trs aspas (""").

Uma docstring deve resumir clara e concisamente o objetivo da classe ou mtodo que est descrevendo. Deve explicar quaisquer parmetros cujo uso no seja imediatamente bvio e tambm um bom lugar para incluir exemplos curtos de como usar a API. class Point:
    "Representa um ponto em coordenadas geomtricas bidimensionais."

    def __init__(self, x=0, y=0):
        """Inicializa a posio de um novo ponto. As coordenadas x e y
           podem ser especificadas. Se no forem, o
           ponto assume a origem."""
        self.move(x, y)

    def move(self, x, y):
        "Move o ponto para uma nova localizao no espao 2D."
        self.x = x
        self.y = y

        "Redefine o ponto para a origem geomtrica: 0, 0."
        self.move(0, 0)

    def calculate_distance(self, other_point):

        ponto passado como parmetro.

        Esta funo usa o Teorema de Pitgoras para calcular
        a distncia entre os dois pontos. A distncia 
        retornada como um nmero float."""

        return (
            (self.x - other_point.x) ** 2
            + (self.y - other_point.y) ** 2
        ) ** (1/2)

Mdulos e pacotes

Em programas pequenos, colocamos nossas classes e funes em um arquivo e um pouco de script no final do arquivo. Conforme nosso projeto cresce, organizamos o cdigo em mdulos, que so simplesmente arquivos Python. Se um mdulo precisa de um recurso de outro mdulo, ele deve importar esse recurso. Um recurso pode ser uma classe, mtodo ou varivel.

Por exemplo, em um sistema de e-commerce, poderamos ter o mdulo database.py contendo classes e funes relacionadas ao banco de dados. Outro mdulo, chamado products.py, responsvel por operaes com produtos, poderia usar a classe Database de database.py atravs de uma das seguintes alternativas:

  • import database
db = database.Database()
# Do queries on db
            
  • from database import Database
db = Database()
# Do queries on db
            
  • from database import Database as DB
db = DB()
# Do queries on db
            
  • from database import Database, Query
db = Database()
q = Query()
# Do queries on db with q

Essas variaes permitem que importemos: o mdulo inteiro; recurso(s) especfico(s); ou recurso(s) renomeado(s) para evitar conflitos de namespace.

Voc pode importar todos os recursos de um mdulo com from database import *, mas isso no recomendado. Isso dificulta a identificao de onde um recurso veio, aumenta a possibilidade de conflitos de nomes com recursos que voc nem vai utilizar e, pior, tambm importa as importaes indiretas do mdulo importado.

Organizando mdulos

Conforme um projeto cresce, til organizar mdulos em pacotes, que so pastas que agregam vrios mdulos, ou at mesmo outros pacotes, dentro delas. Para que uma pasta seja considerada um pacote, voc deve colocar um arquivo __init__.py dentro dela, caso contrrio, no ser possvel importar mdulos dessa pasta.

Suponha a seguinte estrutura de pastas:

root/
    main.py
    ecommerce/
        __init__.py
        database.py
        products.py
        payments/
            __init__.py
            square.py
            stripe.py

Existem dois modos de importao: importao absoluta e importao relativa.

Importaes absolutas

Importaes absolutas especificam o caminho completo para o recurso que est sendo importado. Por exemplo, para importar a classe Product do mdulo products no pacote ecommerce, poderamos usar qualquer uma das seguintes sintaxes:

  • import ecommerce.products
produto = ecommerce.products.Product()
  • from ecommerce import products
produto = products.Product()
  • from ecommerce.products import Product
produto = Product()

Essas instrues funcionaro de qualquer mdulo. Entretanto, especialmente ao importar de uma pasta pai, voc precisa executar o Python da pasta raiz do projeto com a opo -m. Por exemplo, supondo que voc esteja na pasta root, para executar square.py:

python3 -m ecommerce.payments.square

Voc tambm pode carregar pacotes da pasta de pacotes instalados do Python ou da varivel de ambiente PYTHONPATH.

A escolha da sintaxe depende do gosto pessoal e das necessidades especficas do aplicativo. Se muitas classes ou funes de um mdulo forem necessrias, pode ser til importar o mdulo inteiro. Se apenas algumas classes ou funes forem necessrias, elas podem ser importadas diretamente.

Importaes relativas

Voc tambm pode importar recursos a partir do caminho atual, parecido com a navegao de pastas pelo terminal. Basta comear a importao com um ponto, que representa o pacote atual. Por exemplo, para importar Database de dentro de products.py:

from .database import Database

Cada ponto adicional acessa a prxima pasta pai. Por exemplo, para importar Database de dentro de square.py:

from ..database import Database

Tambm possvel importar cdigo diretamente de pacotes usando o arquivo __init__.py. Esse arquivo pode conter recursos disponveis como parte do pacote. Por exemplo, se o arquivo ecommerce/__init__.py contiver a linha from .database import db, poderamos acessar o atributo db usando a seguinte importao: from ecommerce import db.

No entanto, geralmente recomendado evitar colocar muito cdigo no arquivo __init__.py, pois isso pode dificultar a localizao de onde o cdigo declarado e pode levar a comportamentos inesperados.

Organizando o contedo do mdulo

Dentro de um mdulo, podemos especificar variveis, classes ou funes. Essas podem ser usadas para armazenar estados globais sem conflitos de namespace. Por exemplo, poderamos criar um nico objeto de banco de dados disponvel globalmente no mdulo de banco de dados, definindo uma classe Database e instanciando-a no nvel do mdulo:

class Database:
    # implementao do banco de dados
    pass

database = Database()

Esse objeto pode ento ser importado e usado em outros mdulos com from ecommerce.database import database. No entanto, essa abordagem pode ter desvantagens, pois o objeto do banco de dados criado imediatamente quando o mdulo importado, o que pode nem sempre ser desejvel (inicializao lenta ou informaes indisponveis). Para adiar a criao do banco de dados at que ele seja realmente necessrio, poderamos definir uma funo initialize_database:

class Database:
    # implementao do banco de dados
    pass

database = None

def initialize_database():
    global database
    database = Database()

A palavra-chave global usada para especificar que a varivel database dentro da funo se refere varivel de nvel de mdulo. Isso nos permite adiar a criao do banco de dados at que ele seja realmente necessrio.

Todo o cdigo de nvel de mdulo executado imediatamente quando o mdulo importado. No entanto, o cdigo dentro de uma funo ou mtodo no executado at que a funo seja chamada. Isso pode ser til para organizar o cdigo de inicializao e evitar a execuo no intencional ao importar mdulos.

Para evitar a execuo do cdigo de inicializao ao importar um mdulo, recomenda-se colocar o cdigo de inicializao em uma funo (geralmente chamada main) e executar essa funo apenas se o mdulo est sendo executado como um script:

class UsefulClass:
    """Esta classe pode ser til para outros mdulos."""
    pass

def main():
    """Cria uma classe til e faz algo com ela para o nosso mdulo."""
    util = UsefulClass()
    print(util)

if __name__ == "__main__":
    main()

A varivel especial __name__ especifica o nome do mdulo quando ele importado. Quando o mdulo executado como um script, seu valor definido como "__main__".

Tipicamente, mtodos vo em classes, que vo em mdulos, que vo em pacotes. Mas em Python, recursos podem ser definidos em qualquer lugar, inclusive dentro de funes ou mtodos. Isso pode ser til para criar classes temporrias que so necessrias apenas em um escopo especfico:

def format_string(string, formatter=None):
    """Formata uma string usando o objeto formatter, que
    espera ter um mtodo format() que aceite uma string."""

    class DefaultFormatter:
        """Formata uma string em maisculas."""

        def format(self, string):
            return str(string).title()

    if not formatter:
        formatter = DefaultFormatter()

    return formatter.format(string)

hello_string = "hello world, how are you today?"
print(" input: " + hello_string)
print("output: " + format_string(hello_string))

No entanto, essa tcnica no comumente usada em Python.

Quem pode acessar meus dados?

O Python no tem um controle de acesso estrito como algumas outras linguagens de programao orientadas a objetos (public, package, protected, private). Em vez disso, ele depende de convenes e boas prticas para indicar quais mtodos e atributos devem ser considerados privados (ou internos). Por conveno, recursos internos so prefixados com um caractere de sublinhado (_). Ou voc pode escrever docstrings claras indicando isso. No entanto, no h nada no interpretador que impea o acesso direto a eles.

Voc tambm pode prefixar identificadores com dois sublinhados. O Python ir realizar o name mangling, que a mudana de nome desses identificadores, tornando mais difcil acess-los:

class SecretString:
    """Uma maneira no muito segura de armazenar uma string secreta."""

    def __init__(self, string_normal, senha):
        self.__string_normal = string_normal
        self.__frase_senha = senha

    def descriptografar(self, senha):
        """Somente mostra a string se a senha estiver correta."""
        if senha == self.__senha:
            return self.__string_normal
        else:
            return ""

Quando usamos um duplo sublinhado, o recurso prefixado com _<nomedaclasse>. Quando os mtodos dentro da classe acessam a varivel internamente, ela automaticamente 'unmangled'. Quando classes externas desejam acess-la, elas precisam fazer o name mangling manualmente. Portanto, o name mangling no garante privacidade; ele apenas recomenda fortemente.

Para ilustrar os conceitos, vamos criar um aplicativo simples de bloco de notas para linha de comando.

Comearemos com uma anlise bsica: o aplicativo gerenciar pequenas anotaes chamadas notas, armazenadas em um bloco de notas. Cada nota inclui a data de criao e tags (etiquetas) opcionais para fcil categorizao. Os usurios devem ser capazes de editar, pesquisar e modificar as notas.

Os objetos-chave incluem o objeto Note para notas individuais e um objeto continer NoteBook. Embora datas e etiquetas pudessem ser tratadas como objetos, para este prottipo, podemos usar a biblioteca padro do Python para datas e usar uma string separada por vrgulas para etiquetas.

Os objetos Note possuem atributos memo, tags e creation_date. Cada nota tambm requer um id para seleo do usurio em um menu. Para facilitar a pesquisa, um mtodo match no objeto Note pode determinar se uma nota corresponde a uma determinada string, centralizando ajustes relacionados pesquisa.

O objeto NoteBook contm uma lista de notas como atributo e requer um mtodo search para pesquisas filtradas.

A interao com esses objetos ocorre atravs de um menu de linha de comando. Podemos projetar a interface j pensando em acomodar futuras interfaces como GUI ou web.

A classe NoteBook deve incluir mtodos para essas interfaces. Precisaremos de funes para adicionar novas notas, modificar notas existentes por ID e realizar pesquisas. Listar todas as notas pode ser alcanado acessando diretamente o atributo da lista de notas.

A implementao da classe Note pode ficar assim:

import datetime

# Store the next available id for all new notes
last_id = 0


class Note:
    """Represent a note in the notebook. Match against a
    string in searches and store tags for each note."""

    def __init__(self, memo, tags=""):
        """initialize a note with memo and optional
        space-separated tags. Automatically set the note's
        creation date and a unique id."""
        self.memo = memo
        self.tags = tags
        self.creation_date = datetime.date.today()
        global last_id
        last_id += 1
        self.id = last_id

    def match(self, filter):
        """Determine if this note matches the filter
        text. Return True if it matches, False otherwise.

        Search is case sensitive and matches both text and
        tags."""
        return filter in self.memo or filter in self.tags

A implementao da classe NoteBook pode ficar assim:

class Notebook:
    """Represent a collection of notes that can be tagged,
    modified, and searched."""

    def __init__(self):
        """Initialize a notebook with an empty list."""
        self.notes = []

    def new_note(self, memo, tags=""):
        """Create a new note and add it to the list."""
        self.notes.append(Note(memo, tags))

    def modify_memo(self, note_id, memo):
        """Find the note with the given id and change its
        memo to the given value."""
        for note in self.notes:
            if note.id == note_id:
                note.memo = memo
                break

    def modify_tags(self, note_id, tags):
        """Find the note with the given id and change its
        tags to the given value."""
        for note in self.notes:
            if note.id == note_id:
                note.tags = tags
                break

    def search(self, filter):
        """Find all notes that match the given filter
        string."""
        return [note for note in self.notes if note.match(filter)]

Mtodos e variveis, de instncia e estticos

Mtodos e variveis de instncia:

Instncias so sinnimos de objetos. Mtodos de instncia so aqueles que operam em instncias da classe e tm acesso aos atributos especficos de uma nica instncia. Eles so definidos com o primeiro argumento, convencionalmente declarado como self, se referindo prpria instncia. Mtodos de instncia so sinnimos de mtodos. Atributos so sinnimos de variveis de instncia ou de campos. Alguns ainda consideram atributos sinnimos de propriedades, mas este termo mais utilizado no sentido de atributos somente-leitura.

class Dog:
        def __init__(self, name):
            self.name = name

        def bark(self):
            print(f"{self.name} is barking!")

    dog1 = Dog("Buddy")
    dog2 = Dog("Charlie")

    dog1.bark()  # Output: Buddy is barking!
    dog2.bark()  # Output: Charlie is barking!

Nesse exemplo, bark um mtodo de instncia que opera em instncias especficas de Dog.

Mtodos estticos e variveis de classe

Variveis de classe agem como se fossem varivel globais, s que esto associadas classe. Assim como cada instncia tem sua prpria varivel de instncia com seu prprio valor, cada classe tem sua prpria varivel de classe, com seu prprio valor.

Semelhantemente, mtodos estticos agem como se fossem funes. Em vez de serem declarados no nvel de mdulo, so declarados no nvel de classe. Eles podem ser decorados com @staticmethod e no recebem argumentos especiais como self. Em algumas linguagens de programao, mtodos estticos tambm so chamados de mtodos de classe, mas no Python, mtodos de classe tm a ver com outra coisa, que veremos mais para frente.

Voc opta por variveis de classe em vez de variveis globais quando essas variveis esto relacionadas apenas classe em que foram declaradas. So um meio de organizar seu cdigo encapsulando dados dentro da classe, de modo que para serem acessveis, voc deve especificar o nome da classe. Elas so frequentemente utilizadas para armazenar constantes, dados ou configuraes relacionadas exclusivamente classe ou dados compartilhados entre instncias desta classe.

Por outro lado, as variveis globais esto relacionadas ao mdulo como um todo, o que inclui todas as classes que possam ter sido declaradas dentro desse mdulo. Geralmente, variveis globais so usadas para definir constantes ou configuraes que so aplicveis a diversas classes ou funes em um mdulo, ou para armazenar dados que precisam ser acessados globalmente. Entretanto, recomendvel limitar o escopo de suas variveis, evitando o uso excessivo de variveis globais, o que pode tornar seu cdigo menos organizado.

Da mesma forma, os mtodos estticos so a alternativa s funes quando se trata de realizar aes especficas relacionadas apenas a uma classe. Eles so teis para funes utilitrias associadas a uma classe, mas que no requerem acesso a dados especficos de instncias, operaes que no dependem do estado de uma instncia e funes que se encaixam logicamente na estrutura da classe.

J funes regulares so independentes de classes. So ideais como funes utilitrias de uso geral, no diretamente vinculadas a uma classe especfica, funes que executam operaes envolvendo dados de diversas fontes ou funes usadas em diferentes mdulos ou classes.

Em resumo, use mtodos estticos quando tiver funcionalidade intimamente associada a uma classe e use funes regulares quando precisar de funcionalidade independente, reutilizvel e acessvel globalmente. A escolha depende do contexto e do design do seu cdigo.

class Date(object):
    num_instances = 0

    def __init__(self, year=0, month=0, day=0):
        self.day = day
        self.month = month
        self.year = year
        Date.num_instances += 1

    def is_leap_year(self):
        return self.year % 4 == 0 and self.year % 100 != 0 or self.year % 400 == 0

    @staticmethod
    def from_string(date_as_string):
        list = [int(x) for x in date_as_string.split("-")]
        date = Date(list[0], list[1], list[2])
        return date

    def is_date_valid(date_as_string):
        list = [int(x) for x in date_as_string.split("-")]
        return list[0] <= 3999 and list[1] <= 12 and list[2] <= 31


date1 = Date(2100, 12, 25)
print(date1.day, date1.month, date1.year, date1.is_leap_year())
print("Number of instances:", Date.num_instances)
date_string = "2024-02-29"
if Date.is_date_valid(date_string):
    date2 = Date.from_string(date_string)
    print(date2.day, date2.month, date2.year, date2.is_leap_year())
print("Number of instances:", Date.num_instances)

Variveis de classe vs variveis de instncia vs variveis globais

Voc pode importar e utilizar variveis globais e funes diretamente de outros mdulos. Mas para utilizar variveis de classe e mtodos estticos, voc deve especificar o nome da classe antes. Isso lhe d um maior controle sobre o escopo dessas variveis e mtodos, garantindo uma organizao mais eficaz do seu cdigo.

Considere o mdulo test com o seguinte cdigo:

global_var = "a global variable"


def func(par):  # Function
    print(global_var, "and", par)


class Test:
    class_var = "a class variable"  # Class variable

    def __init__(self, par):
        self.inst_var = par  # Instance variable

    def test1(self, par):  # Instance method
        print(self.inst_var, "and", par)

    @staticmethod
    def test3(par):  # Static method
        print(Test.class_var, "and", par)

    def test4(par):  # Static method
        print(Test.class_var, "and", par)

A nica diferena entre usar ou no a decorao @staticmethod, que sem ela voc no pode chamar um mtodo esttico atravs de uma instncia. Como no recomendvel chamar mtodos estticos e mtodos de classe atravs de instncias, ento essa decorao pode ser omitida.

Considere o mdulo main com o seguinte cdigo:

from test import Test
# from test.Test import test2 # error!
from test import func
from test import global_var


def invoking_methods():
    func("a argument")

    t = Test("a instance variable")
    t.test1("a argument")
    t.test2("a argument")

    Test.test2("other argument")
    Test.test3("other argument")

    # t.test3("cannot do this")  # error!
    # Test.test1("this is also forbidden")  # error!

A funo invoking_methods chama cada um dos mtodos atravs de uma instncia e atravs de uma classe.

Observe as linhas 2, 17 e 18 comentadas. No possvel importar mtodos estticos nem variveis de classe diretamente, no possvel chamar um mtodo esttico no decorado atravs de uma instncia, e no possvel chamar um mtodo de instncia atravs de uma classe.

Considere o seguinte trecho de cdigo adicionado ao mdulo main:

def accessing_variables():
    t1 = Test("attribute 1")
    t2 = Test("attribute 2")

    print(t1.inst_var, "|", t1.class_var)
    print(t2.inst_var, "|", t2.class_var)
    print(Test.class_var, "|", global_var)

    t1.inst_var = "attribute 1 modified"
    t1.class_var = "new attribute"
    Test.class_var = "class var modified"
    # global_var = "a local var" # error!

    print(t1.inst_var, "|", t1.class_var)
    print(t2.inst_var, "|", t2.class_var)
    print(Test.class_var, "|", global_var)

    # print(Test.inst_var) # error!

A funo accessing_variables acessa variveis de instncia e de classe por meio de instncias e da classe, alm de acessar uma varivel global.

Observe a linha 12 comentada. Se uma funo ou mtodo no possuir uma varivel local com esse nome, ela procura acessar a varivel global. Mas no possvel modificar variveis globais sem antes especific-las como globais usando a palavra-chave global. Se tentar fazer isso, na verdade, o Python interpretar como uma tentativa de criar uma nova varivel local. Portanto, no recomendvel acessar variveis globais sem antes especific-las como globais.

Agora observe a linha 18 comentada. Uma classe no consegue acessar variveis de instncias. J uma instncia consegue acessar as variveis de sua classe como se fossem seus atributos. Se uma instncia no possuir um atributo com esse nome, ela procura acessar a varivel de classe. Mas no possvel modificar variveis de classe atravs da instncia. Se tentar fazer isso, na verdade, estar criando um novo atributo. Portanto, no recomendvel acessar variveis de classe atravs de instncias.

O que so objetos?

Em Python, tudo objeto. Mas o que um objeto? Linguagens de programao diferentes definem "objeto" de maneiras diversas. Em algumas, objetos so aqueles que possuem atributos e mtodos; em outras, so aqueles que podem ser herdados.

No Python, a definio mais flexvel. Alguns objetos no tm atributos ou mtodos, e nem todos os objetos podem ser herdados. No entanto, tudo objeto no sentido de que pode ser atribudo a uma varivel ou passado como argumento para uma funo. Portanto, funes, classes e nmeros so objetos! Por exemplo, todas as funes tm o atributo doc, que retorna a doc-string definida no cdigo-fonte da funo.

No Python, todo objeto possui o atributo __dict__, que um dicionrio contendo os atributos do objeto e seus valores:

class Test:
    """uma classe de teste"""

    class_var = 10

    def test(self):
        self.inst_var = 20


def func(par):
    """uma funo que imprime os atributos `__doc__` e `__dict__` do parmetro `par`"""
    print("\n", par.__doc__, par.__dict__)


t = Test()
t.test()
func(t)
func(Test)
func(func)
