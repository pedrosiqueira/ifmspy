UVa 11734 - Grande Número de Times Resolverá Isto

Escrever código em uma competição de programação para um problema fácil não é uma tarefa difícil, mas os competidores inexperientes não sabem que sua saída deve corresponder exatamente à dos juízes. Muitas vezes, eles adicionam espaços extras em sua saída, porque acreditam que isso melhora a apresentação. No entanto, a exigência real é que a saída deve corresponder caractere por caractere à dos juízes. Execuções que apresentam espaços adicionais recebem o veredito ‘Output Format Error’. Se a saída ainda não corresponder após ignorar os espaços, o veredito é ‘Wrong Answer’. Um veredito de ‘Yes’ é dado quando a saída corresponde exatamente. Neste problema, você terá que determinar o veredito para as execuções dadas, considerando a saída do time e a dos juízes.

Entrada

A primeira linha da entrada consiste em um número inteiro positivo t < 20, onde t denota o número de casos de teste. Cada caso consiste em duas linhas. A primeira linha é a saída do time e a segunda linha é a saída dos juízes. Cada linha contém pelo menos um e no máximo 20 caracteres. A saída dos times consiste em letras e espaços. A saída dos juízes consiste apenas de letras.

Saída

Para cada caso de entrada, haverá uma linha de saída. Ela conterá primeiro o número do caso seguido pelo veredito. Veja a saída de exemplo para o formato exato.

Exemplo de Entrada/Saída

yes
yes
Casematters
casematters
no space please
nospaceplease
 
Case 1: Yes
Case 2: Wrong Answer
